CHAPTER 16 / 20 · MODULE D

圖與網路探勘

蛋白質互動網路、基因調控、社交網路、藥物-靶點——關係本身就是訊號。

PPI networks, gene regulation, social graphs, drug-target — relationships themselves are the signal.

概覽

什麼是圖？

圖 (Graph) G=(V,E) 由 節點 (V) 與 邊 (E) 組成。依結構分：(1) 有向 vs 無向（Twitter follow 有向，Facebook 朋友無向）、(2) 加權 vs 無加權（蛋白質互動置信度）、(3) 同質 vs 異質（藥物–基因–疾病多型態 KG）、(4) 靜態 vs 動態 (temporal)。

圖探勘任務：中心性（找重要節點）、社群偵測（cluster 模組）、節點嵌入（轉成向量做下游 ML）、連結預測 (link prediction)、知識圖譜推理。生醫應用：藥物再利用 (drug repurposing)、副作用預測、致病基因排序。

A graph G=(V,E) consists of nodes (V) and edges (E). Structurally: (1) directed vs undirected (Twitter follow vs Facebook friend), (2) weighted vs unweighted (PPI confidence), (3) homogeneous vs heterogeneous (drug–gene–disease KG), (4) static vs dynamic / temporal.

Tasks: centrality (important nodes), community detection (modules), node embedding (vectors for downstream ML), link prediction, knowledge-graph reasoning. Biomedical use: drug repurposing, side-effect prediction, disease-gene ranking.

中心性

一、四種重要的中心性

🔢

① Degree

節點鄰居數。最簡單：人脈廣的人；連結最多的蛋白。

Number of neighbors. Simplest: well-connected person; high-degree protein.

🌉

② Betweenness

該節點落在多少對最短路徑上。「橋樑」型：交通樞紐、跨社群中介者。

Fraction of shortest paths passing through. "Bridge" nodes: hubs, cross-community brokers.

⭐

③ Eigenvector

「重要鄰居讓你也重要」。是鄰接矩陣最大特徵值對應的特徵向量。

"Important neighbors make you important". Eigenvector of the largest eigenvalue of the adjacency matrix.

🏆

④ PageRank

有向圖版的 eigenvector + 「random surfer + 阻尼 d」。PR(v) = (1−d)/N + d · Σ PR(u)/L(u)。當 d=0.85 是經典設定。

Directed-graph version + "random surfer + damping d". PR(v) = (1−d)/N + d · Σ PR(u)/L(u). d=0.85 is classic.

⏱️

⑤ Closeness

到所有其他節點最短路徑的倒數平均。值高=「離大家都近」。

Inverse mean shortest-path to all others. High = "close to everyone".

🔄

⑥ HITS

Kleinberg：好的 hub 連到很多 authority，好的 authority 被很多 hub 連。雙向迭代。

Kleinberg: good hubs link to many authorities; good authorities are linked by many hubs. Mutually iterative.

Leiden adds a refinement + fast local move that guarantees connected communities and converges better. Now default in Scanpy and Seurat v5.
GNN 與 KG

三、GNN 與生醫知識圖譜

GNN (Graph Neural Network) 透過 message passing：每層中每個節點把自己 + 鄰居的特徵 aggregate → transform → 更新隱藏狀態。GCN (Kipf 2017)、GAT (attention)、GraphSAGE (sampling) 是三大基本款；GIN 在分子預測上常見。

生醫知識圖譜 (Biomedical KG) 將藥、基因、蛋白、疾病、副作用、文獻等串成異質網路：Hetionet (Himmelstein 2017, 47K nodes / 2M edges, 11 types)、DisGeNET (gene-disease)、PrimeKG (2023, 4M+ relations) 用於藥物再利用、罕病基因排序。

GNN = message passing: at each layer, every node aggregates self + neighbor features → transform → update hidden state. GCN (Kipf 2017), GAT (attention), GraphSAGE (sampling) are the three classics; GIN dominates molecular tasks.

Biomedical KGs connect drug-gene-protein-disease-side effect-literature as heterogeneous networks: Hetionet (Himmelstein 2017, 47K nodes / 2M edges, 11 types), DisGeNET (gene-disease), PrimeKG (2023, 4M+ relations) — used for drug repurposing, disease-gene ranking.

決策樹

四、如何選擇圖探勘方法？

🌳 圖探勘決策樹

Q1:
找 重要節點？ → Degree / Betweenness / PageRank / Eigenvector
Q2:
找 社群 / 模組？ → Leiden > Louvain（前者保證連通）
Q3:
要把節點變成 向量 做下游 ML？ → node2vec (BFS/DFS 可調) 或 DeepWalk
Q4:
有 節點特徵 且要可歸納到新節點？ → GraphSAGE / GAT (PyG, DGL)
Q5:
異質圖（多種節點/邊）？ → KG 嵌入 (TransE, RotatE) 或 RGCN
Q6:
要 連結預測（推薦、藥物再利用）？ → GNN + 邊分類器 / KG 三元組 score
Q1:
Find important nodes? → Degree / Betweenness / PageRank / Eigenvector
Q2:
Detect communities? → Leiden > Louvain (the former guarantees connectivity)
Q3:
Need node vectors for downstream ML? → node2vec (BFS/DFS tunable) or DeepWalk
Q4:
Have node features, need inductive on new nodes? → GraphSAGE / GAT (PyG, DGL)
Q5:
Heterogeneous graph (multi-type nodes/edges)? → KG embedding (TransE, RotatE) or RGCN
Q6:
Link prediction (recommendation, drug repurposing)? → GNN + edge classifier / KG triple score

實作：PageRank + Leiden + node2vec + GNN

library(igraph); library(tidygraph); library(ggraph)

# 1) 建圖 / Create graph
g <- graph_from_data_frame(edges_df, directed = TRUE)

# 2) 中心性 / Centralities
deg <- degree(g, mode = "in")
btw <- betweenness(g, normalized = TRUE)
pr  <- page_rank(g, damping = 0.85)$vector

# 3) 社群偵測 — Leiden（首選）
clu <- igraph::cluster_leiden(g, objective_function = "modularity",
                                resolution_parameter = 1)
V(g)$comm <- membership(clu)

# 4) 視覺化 / Plot
ggraph(as_tbl_graph(g), layout = "fr") +
  geom_edge_link(alpha = .3) +
  geom_node_point(aes(size = pr, color = factor(comm))) +
  geom_node_text(aes(label = name), repel = TRUE)

class GCN(torch.nn.Module):
    def __init__(self, in_d, h, out_d):
        super().__init__()
        self.c1 = GCNConv(in_d, h); self.c2 = GCNConv(h, out_d)
    def forward(self, x, edge_index):
        x = self.c1(x, edge_index).relu()
        return self.c2(x, edge_index)
