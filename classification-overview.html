分類概論：KNN 與 Logistic Regression

分類是監督式學習的基石。從最直觀的 KNN「物以類聚」，到 Logistic Regression 的機率輸出與可解釋係數——兩者分別代表 lazy learner 與 parametric model 的兩端。

Classification is the cornerstone of supervised learning. From the intuitive "birds of a feather" KNN to Logistic Regression's probabilistic outputs and interpretable coefficients — they represent the two ends of the lazy-vs-parametric spectrum.

概覽

什麼是分類？

分類 (classification) 是給定特徵 X 預測離散標籤 y ∈ {0,1,…,K-1} 的監督式學習任務。二元分類（如糖尿病/非糖尿病）最常見，多元分類（如 ICD 章節）需要 softmax 或 one-vs-rest 策略。

本章聚焦兩個「入門即經典」的方法：KNN（無模型、依賴距離）與 Logistic Regression（線性參數模型、輸出機率）。它們分別示範了「記憶」與「歸納」兩種根本不同的學習哲學。

Classification is the supervised task of predicting a discrete label y ∈ {0,1,…,K-1} from features X. Binary tasks (e.g., diabetic vs not) are the most common; multiclass tasks (e.g., ICD chapter) require softmax or one-vs-rest strategies.

This chapter focuses on two "entry-level classics": KNN (non-parametric, distance-based) and Logistic Regression (parametric, probabilistic). They embody two fundamentally different learning philosophies — memorize vs generalize.

KNN

一、K-Nearest Neighbors

🧭

直覺

查詢點的類別 = 最近 K 個鄰居的多數投票。沒有「訓練」階段，模型 = 全部訓練資料。

The query label = majority vote among the K closest neighbors. No "training" phase — the model IS the training data.

🎯

K 的取捨

K 小 → 低偏差高變異（過擬合、易受雜訊影響）；K 大 → 高偏差低變異（過平滑）。常用 √n 或交叉驗證。

Small K → low bias, high variance (overfits noise); large K → high bias, low variance (oversmoothes). Use √n or CV.

⚠️

陷阱

必須先 scaling；高維時失效（curse of dimensionality）；類別不平衡時投票傾向多數類，需 weighted KNN。

Must scale first; fails in high-D (curse of dimensionality); class imbalance biases votes — use weighted KNN.

💡
距離加權：weighted KNN 將鄰居以 1/d 加權，比單純多數投票更穩健，sklearn 設 weights='distance' 即可。 Distance weighting: Weighted KNN votes by 1/d — more robust than plain majority. In sklearn, set weights='distance'.
互動模擬

KNN 決策邊界：拖動 K 觀察變化

在二維玩具資料上跑 KNN——K 小時邊界凹凸不平（過擬合），K 大時邊界平滑但會誤分稀有點。

Run KNN on a 2D toy dataset — small K gives jagged boundaries (overfit), large K oversmoothes and misclassifies rare points.

Logistic Regression

二、邏輯迴歸

線性迴歸輸出實數，但分類需要 [0,1] 機率。Logistic Regression 用 sigmoid σ(z) = 1/(1+e-z) 把線性組合 z = β₀ + β·x 壓到 [0,1]。對 odds 取對數即得線性形式：log(p/(1-p)) = β·x，因此係數 βj 的解釋是「xj 增加 1 單位，log-odds 增加 βj」或「odds 乘以 eβj」。

多元分類用 Softmax：p(y=k) = ezk / Σ ezj，等同於 K-way Logistic Regression。

Linear regression outputs real numbers, but classification needs probabilities in [0,1]. Logistic Regression uses the sigmoid σ(z) = 1/(1+e-z) to squash z = β₀ + β·x into [0,1]. Taking log of the odds gives a linear form: log(p/(1-p)) = β·x. Each βj means: "a unit increase in xj raises log-odds by βj" — i.e., multiplies the odds by eβj.

Multiclass uses Softmax: p(y=k) = ezk / Σ ezj — equivalent to K-way Logistic Regression.

正則化懲罰項效果何時用
L1 (Lasso)λΣ|βj|稀疏解高維特徵選擇
L2 (Ridge)λΣβj²縮小但不歸零多重共線
Elastic NetαL1 + (1-α)L2兼具兩者相關特徵成群
決策樹

三、KNN vs Logistic：如何選？

🌳 分類器選擇決策樹

Q1:
需要機率輸出 + 可解釋係數（臨床決策、論文 OR 表）？ → Logistic Regression
Q2:
決策邊界明顯非線性且樣本充足？ → KNN（或樹型，見 Ch 6）
Q3:
特徵高維且雜訊多（>100 維）？ → 避開 KNN；用 L1-Logistic 或樹
Q4:
類別不平衡（<5% 陽性）？ → class_weight='balanced' 或 SMOTE
Q5:
需要多元分類？ → Softmax / multinomial Logistic
Q1:
Need probability outputs + interpretable coefficients (clinical decisions, OR tables)? → Logistic Regression
Q2:
Decision boundary clearly nonlinear with enough samples? → KNN (or trees, Ch 6)
Q3:
High-D and noisy (>100 features)? → Avoid KNN; use L1-Logistic or trees
Q4:
Class imbalance (<5% positive)? → class_weight='balanced' or SMOTE
Q5:
Need multiclass? → Softmax / multinomial Logistic
⚠️
係數 ≠ 因果：Logistic 係數只是「相關」程度，未控制的混雜會讓 OR 偏移。臨床因果推論請用 Causal Inference (do-calculus, PSM, IPTW)。 Coefficient ≠ causation: Logistic coefficients are associations, not causal effects. Unmeasured confounders distort the OR. For causal claims use causal-inference frameworks (do-calculus, PSM, IPTW).
程式碼

實作：訓練、調 K、看係數

library(class); library(tidymodels)
data(iris); X <- scale(iris[, 1:4])
idx <- sample(nrow(X), 100)

# KNN（class 套件，最直接） / KNN
pred <- class::knn(train=X[idx,], test=X[-idx,], cl=iris$Species[idx], k=5)
mean(pred == iris$Species[-idx])

# tidymodels 統一介面 + CV 調 K / tidymodels with CV
rec  <- recipe(Species ~ ., data=iris) |> step_normalize(all_numeric())
spec <- nearest_neighbor(neighbors=tune()) |> set_engine("kknn") |> set_mode("classification")
wf   <- workflow() |> add_recipe(rec) |> add_model(spec)
res  <- tune_grid(wf, resamples=vfold_cv(iris, v=5), grid=tibble(neighbors=seq(1,25,2)))

# Logistic Regression with L1 (glmnet) / L1-Logistic
library(glmnet)
fit <- cv.glmnet(X[idx,], as.integer(iris$Species[idx]=="versicolor"),
                  family="binomial", alpha=1)  # alpha=1 → Lasso
coef(fit, s="lambda.min")
from sklearn.neighbors import KNeighborsClassifier
from sklearn.linear_model import LogisticRegression
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import cross_val_score, GridSearchCV
from sklearn.pipeline import Pipeline

# KNN with CV on K / KNN + CV
pipe = Pipeline([("sc", StandardScaler()), ("knn", KNeighborsClassifier(weights="distance"))])
gs = GridSearchCV(pipe, {"knn__n_neighbors": range(1,26,2)}, cv=5, scoring="accuracy")
gs.fit(X, y); print(gs.best_params_, gs.best_score_)

# Logistic Regression — L2 / L1 / Elastic Net
lr_l2 = LogisticRegression(penalty="l2", C=1.0, max_iter=1000)
lr_l1 = LogisticRegression(penalty="l1", C=0.5, solver="liblinear")
lr_en = LogisticRegression(penalty="elasticnet", l1_ratio=0.5, solver="saga")

# Multinomial (softmax) for multiclass / softmax
lr_mc = LogisticRegression(multi_class="multinomial", solver="lbfgs", max_iter=1000)
lr_mc.fit(X, y);  print("coefs", lr_mc.coef_.shape)  # (K, n_features)

# Class imbalance / 不平衡資料
lr_bal = LogisticRegression(class_weight="balanced")
