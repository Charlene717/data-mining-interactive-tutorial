模型評估與調參

「模型有多好？」這個問題沒有單一答案。本章涵蓋交叉驗證、混淆矩陣、ROC/PR、校準與三大搜尋策略——一份完整的評估框架。

"How good is the model?" has no single answer. This chapter covers cross-validation, confusion matrix, ROC/PR, calibration, and the three search strategies — a complete evaluation framework.

概覽

為什麼評估這麼難？

單一準確率 (accuracy) 在類別不平衡時極度誤導——若 99% 樣本為陰性，模型「全猜陰性」就 99% 準確率，但對任何陽性都毫無預測力。實務上需要從切資料的方式（CV）、選指標、看分布、做校準、調參五個面向整體評估。

下方表格展示同一個模型在不同指標下可能呈現完全不同樣貌，這就是為什麼「報告單一數字」是 ML 論文的常見錯誤。

A single accuracy number is wildly misleading under class imbalance — if 99% of samples are negative, "always predict negative" hits 99% accuracy with zero predictive power for positives. In practice you must combine splitting strategy (CV), metric choice, distribution, calibration, and tuning for honest evaluation.

The table below shows how the same model can look completely different under different metrics — this is why "report a single number" is one of the most common errors in ML papers.

指標

二、混淆矩陣與衍生指標

✅

Accuracy

(TP+TN)/N。類別平衡時可用，不平衡時誤導。

(TP+TN)/N. OK when balanced; misleading under imbalance.

🎯

Precision / Recall / F1

P=TP/(TP+FP), R=TP/(TP+FN), F1=2PR/(P+R)。F-β 加權 (β>1 重視 recall)。

P=TP/(TP+FP), R=TP/(TP+FN), F1=2PR/(P+R). F-β weights recall (β>1).

📈

ROC-AUC / PR-AUC

ROC：TPR vs FPR；PR：Precision vs Recall。類別不平衡時 PR-AUC 更誠實。

ROC: TPR vs FPR; PR: Precision vs Recall. Under heavy imbalance, PR-AUC is more honest.

💡
校準 (Calibration)：模型輸出 0.8 是否真的 80% 機率事件會發生？Brier score 與 reliability diagram 評估校準度；不準時用 Platt scaling 或 Isotonic regression 校正。 Calibration: If the model outputs 0.8, does the event truly occur 80% of the time? Use Brier score and reliability diagrams; correct with Platt scaling or isotonic regression.

四、要報告什麼指標？

🌳 指標選擇決策樹

Q1:
類別極度不平衡（陽性 <5%）？ → PR-AUC + Recall@k，避免單看 Accuracy / ROC-AUC
Q2:
FN 成本>>FP（漏診癌症）？ → Recall / F-β (β=2) 優先
Q3:
FP 成本>>FN（垃圾訊息）？ → Precision 優先
Q4:
機率本身會被下游用（風險分層、決策）？ → 報告 Brier + reliability diagram
Q5:
多元分類？ → macro-F1 (各類等權) 或 weighted-F1 (按樣本量)
Q6:
調超參時？ → 用 nested CV；外圈報泛化、內圈調 hyperparam
Q1:
Severe imbalance (positives <5%)? → PR-AUC + Recall@k; never Accuracy / ROC-AUC alone
Q2:
FN cost >> FP (missed cancer)? → Prioritize Recall / F-β (β=2)
Q3:
FP cost >> FN (spam)? → Prioritize Precision
Q4:
Downstream uses the probability itself? → Report Brier + reliability diagram
Q5:
Multiclass? → macro-F1 (equal weight) or weighted-F1 (sample-weighted)
Q6:
When tuning hyperparameters? → Use nested CV; outer = generalization, inner = tuning
🚨
調參偏誤：用同一份驗證集既調超參又報 metric → 分數一定偏樂觀。Nested CV 或留一個從未動過的 holdout 是必須。 Tuning bias: Using one validation set to both tune and report → metrics are guaranteed optimistic. Use nested CV or a true never-touched holdout.

inner = StratifiedKFold(5); outer = StratifiedKFold(5)

nested_scores = cross_val_score(gs, X, y, cv=outer, scoring="average_precision")

# 3. Calibration — Platt / Isotonic

# 4. Optuna — Bayesian + pruning
def objective(trial):
    params = {"n_estimators": trial.suggest_int("n_estimators",100,1000),
              "max_depth": trial.suggest_int("max_depth",3,12),
              "learning_rate": trial.suggest_float("lr",.01,.3,log=True)}
    return cross_val_score(xgb.XGBClassifier(**params), X, y, cv=5,
                           scoring="average_precision").mean()
study = optuna.create_study(direction="maximize",
                            pruner=optuna.pruners.HyperbandPruner())
study.optimize(objective, n_trials=100)
