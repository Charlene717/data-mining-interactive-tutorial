樸素貝氏與支持向量機

兩個經典分類器的對照：NB 用機率公式做出「天真但有效」的判斷；SVM 用幾何邊界找最大間隔——並透過 kernel trick 把線性世界擴展到任意維度。

Two classics in contrast: NB makes "naively effective" decisions via probability; SVM finds maximum-margin boundaries — and extends the linear world to any dimension via the kernel trick.

開場故事
情境劇

1998 雷蒙德 · Mehran Sahami

1998 年，Microsoft Research 一位剛博士畢業的 Mehran Sahami 在 AAAI Workshop 發表了一篇 7 頁的論文：A Bayesian Approach to Filtering Junk E-Mail。他用最簡單的 Multinomial Naive Bayes 演算法，配上「特徵 = 出現過某詞」的 bag-of-words，在當時還沒有「垃圾郵件」概念的時代，做出了第一個可用的過濾器。25 年過去，2023 年 Gmail 的 spam filter 引擎中心仍然有 NB classifier 的影子——只是疊上了 Transformer 與 graph 演算法。

同一年的同一個研究機構，另一位俄裔學者 Vladimir Vapnik 剛搬到貝爾實驗室，繼續推進他 1995 年和 Corinna Cortes 一起發表的 Support Vector Machine——這個演算法在 1998 年由 Thorsten Joachims 拿來分類路透社文章，準確率 86%，遠勝當時的 NB。直到 2010 年代深度學習興起，SVM 才退到第二線。

本章把這兩個來自同一年代、解同一問題、哲學完全相反的演算法擺在一起——你會看見「機率框架 (NB)」與「幾何框架 (SVM)」如何形成兩條互補的工具線。

1998. At Microsoft Research, a freshly minted PhD named Mehran Sahami presented a 7-page paper at AAAI Workshop: A Bayesian Approach to Filtering Junk E-Mail. He used the simplest Multinomial Naive Bayes with bag-of-words features — and built the first usable spam filter in an era when "spam" wasn't even a household word. 25 years later, Gmail's spam engine still has an NB classifier inside, beneath stacks of Transformer + graph algorithms.

Same year, same research circles: Vladimir Vapnik had just moved to Bell Labs, continuing the Support Vector Machine he had co-authored with Corinna Cortes in 1995. In 1998, Thorsten Joachims applied SVM to Reuters news classification — 86% accuracy, far beating NB. SVM ruled text and small-N tabular until the deep-learning surge of the 2010s.

This chapter places these two algorithms — same era, same problem, opposite philosophy — side by side. You'll see how the "probabilistic frame (NB)" and the "geometric frame (SVM)" form two complementary tool lines.

NB + SVM 關鍵詞 ×14

Bayes' Theorem
貝氏定理
看到症狀後，得病的機率怎麼更新？
After seeing symptoms, how do we update the chance of disease?
P(y|x) = P(x|y)·P(y)/P(x)；後驗 ∝ 似然 × 先驗。Bayes 1763 遺作。
P(y|x) = P(x|y)·P(y)/P(x); posterior ∝ likelihood × prior. Bayes' 1763 posthumous paper.
Prior / Likelihood / Posterior
先驗 / 似然 / 後驗
先驗 = 看資料前的猜；似然 = 資料的「驚不驚訝」；後驗 = 看資料後的猜。
Prior = guess before data; likelihood = how surprised by the data; posterior = guess after data.
P(y)：類別比例；P(x|y)：類別內生成模型；P(y|x)：分類器輸出。
P(y): class prevalence; P(x|y): class-conditional generative model; P(y|x): classifier output.
Naive Bayes (Conditional Independence)
獨立假設
假裝每個症狀都跟其他症狀無關——天真但好用。
Pretend every symptom is independent of the others — naive but useful.
P(x|y) = Πj P(xj|y)；參數從 O(2p) 降到 O(p)，但忽略相關性。
P(x|y) = Πj P(xj|y); parameters drop from O(2p) to O(p), but ignores correlations.
Laplace Smoothing
Laplace 平滑
沒見過的詞給它 +1 票，避免機率為零。
Give every unseen word +1 vote so probabilities are never zero.
P(xj=v|y) = (count + α) / (Ny + α·|V|)；α=1 是 add-one。
P(xj=v|y) = (count + α) / (Ny + α·|V|); α=1 is add-one.
Gaussian / Multinomial / Bernoulli NB
三種 NB 變體
連續、計數、是否出現——三種 P(x|y) 對應三種特徵。
Continuous, count, presence — three P(x|y) for three feature types.
Gaussian: 𝒩(μy,σy²); Multinomial: 多項分布 (詞頻); Bernoulli: 二元 (詞出現)。
Gaussian: 𝒩(μy,σy²); Multinomial: multinomial dist (term-freq); Bernoulli: binary (term presence).
MAP Estimate
最大後驗
挑出後驗機率最大的那個類別作為預測。
Pick the class with the highest posterior probability.
ŷ = argmaxy P(y|x) = argmaxy log P(y) + Σj log P(xj|y)。實際用 log-likelihood 避免下溢。
ŷ = argmaxy P(y|x) = argmaxy log P(y) + Σj log P(xj|y). Use log-likelihood to avoid underflow.
Support Vector Machine
支持向量機
兩個國家之間蓋最寬的高速公路，城牆 (support vectors) 決定路寬。
Build the widest highway between two countries; only the border cities (support vectors) decide the width.
min ½‖w‖² s.t. yi(wᵀxi+b)≥1。對偶形式只依 sup. vectors 與核函數 K(x,x′)。
min ½‖w‖² s.t. yi(wᵀxi+b)≥1. Dual form depends only on sup. vectors and kernel K(x,x′).
Max-Margin / Margin
最大間隔
兩類之間留越寬越好——容錯能力更高。
The wider the gap between classes, the more tolerant the model.
margin = 2/‖w‖；最大化即最小化 ‖w‖²，VC 維度上界由 margin 控制 (Vapnik 1995)。
margin = 2/‖w‖; maximising it ≡ minimising ‖w‖². VC dimension bound controlled by margin (Vapnik 1995).
Support Vector
支持向量
真正決定邊界的那幾個樣本——其他都「不重要」。
The few samples that actually decide the boundary — the rest don't matter.
αi>0 的訓練樣本；KKT 條件下只有這些影響 w。
Training samples with αi>0; KKT conditions guarantee only these affect w.
Kernel Trick
核技巧
不用真的爬到高維，只要算 K(x, x′) 就好。
Don't actually climb into high-D — just compute K(x, x′).
Mercer 條件下 K(x,x′)=⟨φ(x),φ(x′)⟩；隱式映射 φ 不必算。
Under Mercer's condition K(x,x′)=⟨φ(x),φ(x′)⟩; the implicit map φ never computed.
RBF / Gaussian Kernel
RBF 核
附近的樣本影響大、遠的影響小——bell shape。
Nearby samples matter a lot, far ones not — bell-shaped.
K(x,x′) = exp(−γ‖x−x′‖²)；γ 大 → 局部曲折；γ 小 → 平滑。
K(x,x′) = exp(−γ‖x−x′‖²); large γ → wiggly; small γ → smooth.
Soft Margin (C)
軟間隔 (C)
允許幾個搗蛋鬼跨界——容忍多少，由 C 決定。
Allow a few troublemakers to cross the line — C controls how many.
min ½‖w‖² + C·Σξi；C 大 → 不容忍 (硬)；C 小 → 容忍 (軟)。等價 1/λ 在 Logistic L2。
min ½‖w‖² + C·Σξi; large C → strict (hard); small C → tolerant (soft). Equivalent 1/λ in L2-Logistic.
Hinge Loss
Hinge 損失
分對且離邊界遠 → 0 損失；其他 → 線性增長。
Right and far from boundary → 0 loss; otherwise linear penalty.
L = max(0, 1 − y·(wᵀx+b))；非平滑、僅在 margin 內非零，造就稀疏 dual。
L = max(0, 1 − y·(wᵀx+b)); non-smooth, nonzero only inside the margin → sparse dual.
SMO (Sequential Minimal Optimization)
SMO 解法
一次只解兩個 α，不斷重複——SVM 訓練的標準作法。
Solve two α's at a time, iterate — the standard SVM trainer.
Platt 1998 演算法；解析解逐對更新，是 LIBSVM 內核。複雜度 ~O(n²)–O(n³)。
Platt's 1998 algorithm; analytic pair-update, the engine inside LIBSVM. Complexity ~O(n²)–O(n³).

類比

天真醫生 vs 高速公路工程師

Naive Bayes 就像「一個天真的全科醫生：他假設『發燒和咳嗽彼此獨立』，於是把每個症狀對流感的影響獨立乘起來——明明知道這不對，但他算得快、判得準、便宜得不得了」。SVM 是「另一個工程師：他不問機率，只問『兩個國家邊界線在哪』。他在地圖上鋪一條最寬的高速公路，路邊的城鎮 (support vectors) 決定路寬。剩下的城鎮在不在路上，他根本不在乎」。一個是機率世界觀，一個是幾何世界觀——同一個問題的兩條解法。

Naive Bayes is like "a naive GP who assumes 'fever and cough are independent', multiplies each symptom's effect on flu independently — knows it's not literally true, but he's fast, accurate, and dirt-cheap." SVM is "the highway engineer who doesn't care about probability — only 'where does the border go?' He lays the widest possible highway on the map; the towns on either edge (support vectors) decide its width. The rest of the towns he doesn't care about." One is a probabilistic worldview, the other geometric — two ways to crack the same problem.

貝氏定理

一、Naive Bayes 的核心

貝氏定理：P(y | x) ∝ P(x | y) · P(y)。要計算後驗，需要聯合機率 P(x₁,…,xp | y)——在高維下完全不可估。Naive Bayes 的「樸素 (naive)」假設是：給定類別 y 後，特徵彼此條件獨立。於是聯合機率拆成 ΠP(xj | y)，參數量從指數降到線性。

即使這假設常常不成立，NB 仍在許多文本/垃圾郵件/醫學初篩任務上表現驚人——因為分類只需要「機率排序正確」而非「機率值正確」。

Bayes' rule: P(y | x) ∝ P(x | y) · P(y). The joint P(x₁,…,xp | y) is impossible to estimate in high dimensions. The "naive" assumption is: features are conditionally independent given y. The joint then factors as ΠP(xj | y) — parameter count drops from exponential to linear.

Even when this assumption is wrong, NB performs remarkably well on text / spam / clinical triage — because classification only needs the ranking to be right, not the calibrated probabilities.

互動模擬

獨立假設對預測的影響

玩具情境：兩個高度相關的特徵 (如 BMI 和體脂率) 同時出現。NB 把它們當獨立 → 重複計算證據，後驗機率被推得過高。切換獨立假設 (on/off) 看後驗機率變化。

Toy scenario: two highly correlated features (e.g., BMI and body-fat) co-occur. NB treats them as independent → double-counts evidence and inflates the posterior. Toggle the independence assumption to compare.

SVM

二、Support Vector Machine

SVM 不問「哪條直線能分開」，而問「哪條直線離兩類最遠」——它找最大間隔 (max margin) 的超平面，只有最靠近邊界的少數樣本（support vectors）決定模型。

Soft margin (C)：真實資料常無法完美分割。C 控制「允許多少分錯」——C 大 → 不容忍誤分（窄間隔、易過擬合）；C 小 → 容忍誤分（寬間隔、較穩健）。

Kernel trick：把資料隱式投影到高維空間，使在原空間線性不可分的問題在新空間線性可分——而不需真的計算高維坐標，只需算核函數 K(x, x′)。RBF、Polynomial、Sigmoid 是最常用的三種核。

SVM does not ask "which line separates the classes?" but "which separator is farthest from both classes?" — it finds the maximum-margin hyperplane. Only the points closest to the boundary (support vectors) define the model.

Soft margin (C): Real data is rarely perfectly separable. C controls how many misclassifications you tolerate. Large C → low tolerance (narrow margin, overfit risk); small C → high tolerance (wider margin, more robust).

Kernel trick: Implicitly project data into a higher-dimensional space where the problem becomes linearly separable — without ever computing those high-D coordinates. We only need K(x, x′). RBF, Polynomial, and Sigmoid are the three most common kernels.

決策樹

三、NB vs SVM：怎麼選？

🌳 NB / SVM 選擇決策樹

Q1:
資料是文本 / 詞頻 / 高維稀疏？ → Multinomial NB 或 Linear SVM
Q2:
樣本量小 (<1000) 且需要快速 baseline？ → Naive Bayes
Q3:
特徵間高度相關（基因表現）？ → 避開 NB；改用 SVM/Logistic/RF
Q4:
決策邊界明顯非線性且樣本中等 (1k–50k)？ → SVM (RBF kernel)
Q5:
樣本超大 (>100k)？ → SVM 太慢，改用 Linear SVM / SGDClassifier / LightGBM
Q1:
Text / TF-IDF / high-D sparse data? → Multinomial NB or Linear SVM
Q2:
Small sample (<1000), need fast baseline? → Naive Bayes
Q3:
Highly correlated features (gene expression)? → Avoid NB; use SVM / Logistic / RF
Q4:
Nonlinear boundary, medium data (1k–50k)? → SVM with RBF kernel
Q5:
Very large sample (>100k)? → SVM too slow; use Linear SVM / SGDClassifier / LightGBM
⚠️
SVM 必須 scaling：RBF kernel 依賴歐式距離，未 scaling 會被大量級特徵主導。先 StandardScaler 再 fit。 SVM requires scaling: RBF kernel relies on Euclidean distance — large-scale features dominate without scaling. StandardScaler first.

紙筆
STEP 1
訓練資料：1000 封郵件，400 spam (40%)、600 ham (60%)。類別先驗：P(spam)=0.40、P(ham)=0.60。取 log → log P(spam)=−0.916、log P(ham)=−0.511。
Training: 1000 emails, 400 spam (40%), 600 ham (60%). Priors: P(spam)=0.40, P(ham)=0.60. log P(spam)=−0.916, log P(ham)=−0.511.
STEP 2
詞頻 (spam 中)："free" 出現 320 次 / spam 總詞數 4000；"money" 80/4000；"now" 200/4000。詞彙表大小 |V|=8000。
Word counts in spam: "free" 320 / total spam words 4000; "money" 80/4000; "now" 200/4000. Vocabulary size |V|=8000.
STEP 3
Laplace 平滑 (α=1)：P("free"|spam) = (320+1)/(4000+8000) = 321/12000 = 0.02675。log = −3.62。同理 P("money"|spam) = 81/12000 = 0.00675 → log = −5.00；P("now"|spam) = 201/12000 = 0.01675 → log = −4.09。
Laplace smoothing (α=1): P("free"|spam) = (320+1)/(4000+8000) = 321/12000 = 0.02675. log = −3.62. Similarly P("money"|spam) = 81/12000 = 0.00675 → log = −5.00; P("now"|spam) = 201/12000 = 0.01675 → log = −4.09.
STEP 4
同樣計算 ham：P("free"|ham) = 21/14000 = 0.0015 → log = −6.50；P("money"|ham) = 11/14000 → log = −7.14；P("now"|ham) = 71/14000 → log = −5.28。
Same for ham: P("free"|ham) = 21/14000 = 0.0015 → log = −6.50; P("money"|ham) = 11/14000 → log = −7.14; P("now"|ham) = 71/14000 → log = −5.28.
STEP 5
查詢「free money now」對 spam：log P(spam|x) ∝ log P(spam) + Σ log P(w|spam) = −0.916 − 3.62 − 5.00 − 4.09 = −13.63。
Query "free money now" vs spam: log P(spam|x) ∝ log P(spam) + Σ log P(w|spam) = −0.916 − 3.62 − 5.00 − 4.09 = −13.63.
STEP 6
同一查詢對 ham：log P(ham|x) ∝ −0.511 − 6.50 − 7.14 − 5.28 = −19.43。
Same query vs ham: log P(ham|x) ∝ −0.511 − 6.50 − 7.14 − 5.28 = −19.43.
STEP 7
結論：log P(spam|x) = −13.63 >> log P(ham|x) = −19.43。判 spam。指數化得 odds ratio = exp(5.80) ≈ 330：spam 機率是 ham 的 330 倍。
Verdict: log P(spam|x) = −13.63 >> log P(ham|x) = −19.43. Predict spam. Exponentiate the diff: odds ratio = exp(5.80) ≈ 330 — spam is 330× more likely.
STEP 8
歸一化：P(spam|x) = exp(−13.63) / (exp(−13.63)+exp(−19.43)) = 1/(1+exp(−5.80)) = 0.9970。注意：因獨立假設常被破壞，這個值可能被高估——實務上跑校準。
Normalise: P(spam|x) = exp(−13.63) / (exp(−13.63)+exp(−19.43)) = 1/(1+exp(−5.80)) = 0.9970. Caveat: the independence assumption is usually violated, so this number is likely over-confident — calibrate in practice.

爭論題

三題爭論

爭論 1

NB 的假設明明錯卻有用？

Domingos-Pazzani 1997：在 Machine Learning 期刊發表經典論文證明：分類器只需要 argmax 排序正確，不需要機率值正確。即使 P(x|y) 因獨立假設失真，只要錯誤方向對所有類別一致，argmax 仍對。所以 NB 即使「機率不準」仍然「分類準」。In Machine Learning: a classifier only needs argmax ordering, not calibrated probabilities. Even when P(x|y) is biased by the independence assumption, if the bias is consistent across classes, argmax still wins. That's why NB "miscalibrates yet classifies well".
實務啟示：想要 NB 的「分類」，沒問題；想要 NB 輸出真實機率，先 Platt scaling 或 isotonic regression 校準。If you need NB's classification, fine; if you need real probabilities, calibrate via Platt or isotonic.
爭論 2

2026 還教 SVM？

挺方：教學價值高——VC dimension、max-margin、kernel trick、convex optimization 都是 ML 基礎概念載體。實務上小 N (<1000) 表格資料 SVM-RBF 仍勝多數樹和 NN (Grinsztajn 2022)。Pedagogical gold — VC dimension, max-margin, kernel trick, convex opt are all foundational. In practice, on small-N (<1000) tabular SVM-RBF still beats most trees and NN (Grinsztajn 2022).
反方：大資料 (n>10⁵) 訓練 O(n²) 太慢；機率輸出需 Platt scaling；超參 (C, γ, kernel) 很難 grid。對 entry-level 工程師而言 XGBoost 學習曲線更平。Big data (n>10⁵): O(n²) too slow; probabilistic output requires Platt scaling; hyperparams (C, γ, kernel) hard to grid. For entry-level engineers XGBoost has a flatter curve.
折衷：教 Linear SVM (hinge loss) 與 RBF SVM 作為「概念載體」，但工程實務不勉強用——重點是學完之後對 LLM 的注意力機制、kernel methods in deep learning (NTK)、Gaussian Process 都更容易理解。Teach Linear SVM (hinge loss) + RBF SVM as "concept vehicles" but don't force them in production. Worth it because the concepts transfer to attention mechanisms, kernel methods in DL (NTK), and Gaussian Processes.
爭論 3

Kernel trick — 聰明還是炒作？

聰明：無需顯式計算高維特徵就能在高維空間做分類；對小資料是「特徵工程的免費午餐」。Mercer 條件給出嚴謹數學基礎。Classify in high-D without explicit feature engineering — "free lunch" on small data. Mercer's condition gives rigorous math foundation.
炒作：大資料時 kernel matrix O(n²) 記憶體爆炸；RBF 對 outlier 敏感；γ/C 互相耦合難調。深度學習出現後，「kernel = feature map」改用學出來的 embedding，更實用。Big data: kernel matrix O(n²) blows up memory; RBF is outlier-sensitive; γ/C coupled and hard to tune. Deep learning replaced "kernel = feature map" with learned embeddings — far more practical.
2024 新意：Neural Tangent Kernel (NTK; Jacot 2018) 把無限寬神經網路與 kernel method 連起來——讓 kernel trick 在深度學習時代重新有研究價值。Neural Tangent Kernel (NTK; Jacot 2018) connects infinitely-wide NNs to kernel methods — bringing the kernel trick back into the deep-learning era.

實案聚光燈

Sahami 1998 — 7 頁定義一代

Mehran Sahami（後成為 Stanford CS 教授）1998 年在 AAAI Workshop 發表 A Bayesian Approach to Filtering Junk E-Mail（共 7 頁）。論文取了 1789 封郵件 (1538 垃圾 + 251 正常)、bag-of-words 特徵、Multinomial NB——測試準確率 92.3%，比當時所有 rule-based filter 都好。文章還討論了「特徵工程的價值」：加入 hand-crafted 特徵 (如 sender ratio、HTML tag) 可再提升 5%。這篇文章為後續所有 e-mail/SMS/評論垃圾過濾建立模板——直到今天 Gmail 的多層 ensemble 仍可看見 NB 的影子。

Mehran Sahami (later Stanford CS professor) presented A Bayesian Approach to Filtering Junk E-Mail at AAAI Workshop 1998 — 7 pages. Trained on 1789 emails (1538 spam + 251 ham), bag-of-words, Multinomial NB — test accuracy 92.3%, beating all rule-based filters of the era. The paper also discussed "the value of feature engineering" — hand-crafted features (sender ratio, HTML tags) added another 5%. This template defined a generation of e-mail / SMS / comment-spam filters. Today's Gmail ensemble still has NB inside.

陷阱

陷阱 5 連發

NB 沒用 log 計算

乘 100 個小機率 → 數值下溢 (underflow) 成 0。務必使用 log-likelihood 加總。

Multiplying 100 tiny probabilities → numerical underflow to 0. Always sum log-likelihoods.

忘記 Laplace 平滑

沒見過的詞 → P(x|y)=0 → 整條後驗歸零。α≥1 的 Laplace 平滑必設。

Unseen word → P(x|y)=0 → entire posterior collapses. Always set α≥1 Laplace.

RBF SVM 沒 scaling

RBF 依賴歐式距離；大尺度特徵主導 ‖x-x'‖²，γ 設定失靈。必須 StandardScaler。

RBF depends on Euclidean distance; large-scale features dominate ‖x-x'‖², γ becomes useless. Must StandardScaler.

SVM 當機率分類器

SVM 預設輸出 decision_function (距離邊界)、不是機率。要機率需 SVC(probability=True) 啟用 Platt scaling，會慢 5×。

SVM defaults to decision_function (distance to boundary), not probability. Need SVC(probability=True) for Platt scaling — 5× slower.

SVM 跑大資料

SVM 訓練 O(n²)–O(n³)，n=10⁵ 時可能跑幾小時。大資料用 LinearSVC + L2 或 SGDClassifier(loss='hinge')。

SVM training O(n²)–O(n³); n=10⁵ may take hours. For big data use LinearSVC + L2 or SGDClassifier(loss='hinge').

壞範例：MultinomialNB() 直接吃 raw counts

MultinomialNB().fit(X, y)——X 是原始 word counts、沒 Laplace 平滑、沒 TF-IDF。對於沒見過的詞會生 P=0；對於常見詞 (the, of) 又會主導預測。

MultinomialNB().fit(X, y) — X is raw word counts, no Laplace, no TF-IDF. Unseen words → P=0; common words (the, of) dominate.

好範例：Pipeline(TfidfVectorizer + MultinomialNB(α=1))

Pipeline([("tfidf",TfidfVectorizer(ngram_range=(1,2), min_df=5, sublinear_tf=True)), ("nb",MultinomialNB(alpha=1.0))])，跑 5-fold CV，回報 PR-AUC + calibrated probabilities。

Pipeline([("tfidf",TfidfVectorizer(ngram_range=(1,2), min_df=5, sublinear_tf=True)), ("nb",MultinomialNB(alpha=1.0))]), 5-fold CV, report PR-AUC + calibrated probs.

進階研究員彩蛋

2024–2026 SOTA

ThunderSVM 0.5 (Wen et al, 2018→2024 update): GPU 加速 SVM，相比 LIBSVM 在 CPU 上快 1000×。提供 sklearn-style API，from thundersvm import SVC。適合 n=10⁵–10⁶ 樣本仍想用 kernel SVM 的場景。

ThunderSVM 0.5 (Wen et al, 2018→2024 update): GPU-accelerated SVM, 1000× over CPU LIBSVM. sklearn-style API: from thundersvm import SVC. Use when n=10⁵–10⁶ and you still want kernel SVM.

🔍 RFF — approximate RBF kernel

Random Fourier Features (Rahimi-Recht NIPS 2007, DOI:10.5555/2981562.2981710) 用 D 個 random projection ωi 把 RBF kernel 近似成 K(x,x′) ≈ z(x)ᵀz(x′)，其中 z(x)=[cos(ωᵀx+b)]/√D。然後 SVM 在 z 上跑 linear SVM——複雜度從 O(n³) 降到 O(nD)。

2014 NIPS Test-of-Time Award 得主。sklearn: from sklearn.kernel_approximation import RBFSampler; rff=RBFSampler(gamma=.5, n_components=500); Xz=rff.fit_transform(X); LinearSVC().fit(Xz, y)。

Random Fourier Features (Rahimi-Recht NIPS 2007, DOI:10.5555/2981562.2981710) approximate RBF kernel as K(x,x′) ≈ z(x)ᵀz(x′) via D random projections; z(x)=[cos(ωᵀx+b)]/√D. Then run linear SVM in z-space — complexity O(n³) → O(nD).

2014 NIPS Test-of-Time Award. sklearn: RBFSampler(gamma=.5, n_components=500) → LinearSVC().

🔍 Bayesian NB — uncertainty in NB

傳統 NB 用 MLE 估參，沒 uncertainty。Bayesian NB 把 P(xj|y) 視為 Beta (或 Dirichlet for multinomial) 分布、加先驗 → 後驗分布，輸出帶不確定性的機率。Witten 2011 + 2024 PyMC 4 提供現成實作。

應用：醫學初篩，當訓練樣本只有 50 筆，傳統 NB 信心過頭；Bayesian NB 輸出 P(disease) = [0.45, 0.85] 信賴區間，給醫師更實用的決策依據。

Vanilla NB uses MLE — no uncertainty. Bayesian NB places Beta (or Dirichlet for multinomial) prior on P(xj|y) → posterior distribution; output probabilities come with uncertainty. Witten 2011 + 2024 PyMC 4 give ready implementations.

Use case: clinical triage with only 50 training samples — vanilla NB over-confident; Bayesian NB gives P(disease) = [0.45, 0.85] credible interval — far more useful for clinician.

🔍 NTK (Jacot et al, NeurIPS 2018)

Neural Tangent Kernel (Jacot-Gabriel-Hongler NeurIPS 2018, arXiv:1806.07572) 證明：無限寬的 NN 在 SGD 訓練下，等價於用一個特定 kernel 跑 kernel regression。這個 kernel 叫 NTK。意義：

  • 解釋了為什麼過度參數化的 NN 仍然能泛化。
  • 連結 SVM/Kernel methods 與 deep learning 的理論。
  • 2024 Tabular NTK (Wang et al) 在小 N 表格上勝過 XGBoost 1.5% AUROC。

Neural Tangent Kernel (Jacot-Gabriel-Hongler NeurIPS 2018, arXiv:1806.07572) proves: an infinitely-wide NN trained by SGD is equivalent to kernel regression with a specific kernel — the NTK. Implications:

  • Explains why over-parameterized NNs still generalize.
  • Connects SVM/kernel methods to deep learning theory.
  • 2024 Tabular NTK (Wang et al.) beats XGBoost by 1.5% AUROC on small-N tabular.
