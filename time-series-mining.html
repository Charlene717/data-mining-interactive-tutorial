時間序列探勘

心率、血糖、空汙、急診人次——所有「隨時間記錄」的資料都是時間序列。

Heart rate, glucose, air quality, ED census — anything "recorded over time" is a time series.

概覽

時序資料的四個關鍵性質

時間序列 = 依時間順序排列的觀測 Xt。與一般機器學習資料的關鍵差異：(1) 有順序性（不能隨機 shuffle）、(2) 有自相關（Xt 依賴 Xt−1）、(3) 有趨勢與季節性、(4) 切割必須 沿時間軸（train 在前、test 在後）。

分析任務涵蓋：預測 (forecasting)、分類 (TSC)、群集 (TSC)、異常偵測、變點偵測 (change-point)、模式發掘 (motif / discord, shapelet, matrix profile)。生醫應用：ICU 早警 (HR, SpO₂)、CGM 血糖預測、ECG 分類、流行病早期偵測。

A time series is a sequence Xt ordered by time. Key differences from i.i.d. data: (1) order matters (no shuffling), (2) autocorrelation (Xt depends on Xt−1), (3) trend + seasonality, (4) splits must be chronological (train precedes test).

Tasks: forecasting, classification (TSC), clustering, anomaly, change-point detection, pattern discovery (motif/discord, shapelet, matrix profile). Biomedical: ICU early warning (HR, SpO₂), CGM glucose prediction, ECG classification, outbreak early detection.

基礎概念

一、平穩性與成分分解

📉

平穩性 Stationarity

均值、變異數、自相關不隨時間改變。多數模型 (ARIMA) 假設平穩；常用 ADF 檢定 (Augmented Dickey-Fuller) 判斷。

Mean, variance, autocorrelation invariant over time. Many models (ARIMA) assume it; tested by ADF (Augmented Dickey-Fuller).

📈

趨勢 Trend

長期方向（上升、下降）。可用 差分 (differencing) 或迴歸殘差去除。

Long-term direction (up/down). Remove with differencing or regression residuals.

🌊

季節性 Seasonality

固定週期（day、week、year）。STL / X-13ARIMA-SEATS 分解。

Fixed-period cycles (daily, weekly, yearly). Decomposed via STL or X-13ARIMA-SEATS.

🔁

自相關 ACF / PACF

ACF：與過去時點的相關；PACF：扣掉中間時點後的偏相關。用來決定 AR、MA 階數。

ACF: correlation with past lags. PACF: partial after controlling intermediates. Guides AR/MA order.

🎲

白噪音

均值=0、變異穩定、無自相關。預測殘差應接近白噪音 → 模型已抓到結構。

Zero mean, constant variance, no autocorrelation. Forecast residuals should look like white noise.

⚠️

變點 Change-point

底層分布突然改變的時刻。PELT、Bayesian Online (BOCPD)、ruptures 套件。

The moment the underlying distribution shifts. PELT, Bayesian Online (BOCPD), ruptures package.

互動模擬

ARIMA(p,d,q) — 三個滑桿，看不同模型行為

p=自迴歸階數 (AR)、d=差分次數、q=移動平均階數 (MA)。調整滑桿，觀察合成序列的趨勢與簡化預測。

p=AR order, d=differencing, q=MA order. Adjust sliders to see synthetic series + a simple-forecast outcome.

決策樹

三、如何挑選時序方法？

🌳 時序方法決策樹

Q1:
需要 預測未來？ → ARIMA / SARIMA / ETS / Prophet / LSTM
Q2:
有 強季節 + 假日效應（如急診人次）？ → Prophet
Q3:
需要 比較兩條序列（不同長度也可）？ → DTW
Q4:
需要 分類時序 且要可解釋？ → Shapelets
Q5:
要找 重複出現的模式 (motif) 或最異常段 (discord)？ → Matrix Profile (STUMPY)
Q6:
要找 結構性變化（中斷、政策介入）？ → Change-point (PELT, BOCPD)
Q1:
Need to forecast? → ARIMA / SARIMA / ETS / Prophet / LSTM
Q2:
Strong seasonality + holidays (e.g., ED census)? → Prophet
Q3:
Compare two series (possibly variable length)? → DTW
Q4:
Classify with interpretability? → Shapelets
Q5:
Find repeating motifs / extreme discords? → Matrix Profile (STUMPY)
Q6:
Detect structural changes (intervention)? → Change-point (PELT, BOCPD)
⚠️
切割勿打亂！時序資料的 train/test 必須按時間順序：早期當訓練、晚期當測試。隨機 K-fold 等於把「未來」洩漏給模型，產生極度樂觀的指標。改用 TimeSeriesSplit 或 rolling-origin / walk-forward。 Do not shuffle! Time-series splits must be chronological — early as train, late as test. Random K-fold leaks "future" data, yielding inflated metrics. Use TimeSeriesSplit or rolling-origin / walk-forward.
程式碼

實作：ARIMA + Prophet + DTW + Matrix Profile

library(forecast); library(fable); library(prophet); library(dtw); library(tseries)

# 1) 平穩性檢定 / Stationarity test (ADF)
adf <- adf.test(y)            # p < 0.05 → reject unit-root → stationary

# 2) 自動 ARIMA 預測 / Auto-ARIMA
fit <- auto.arima(y, seasonal = TRUE, stepwise = FALSE)
fc  <- forecast(fit, h = 24)
autoplot(fc)

# 3) Prophet (含假日、節慶)
df  <- data.frame(ds = dates, y = y)
m   <- prophet(df, yearly.seasonality = TRUE, weekly.seasonality = TRUE)
fut <- make_future_dataframe(m, periods = 90)
pf  <- predict(m, fut)

# 4) DTW 比較兩條 ECG / DTW between two ECG snippets
al <- dtw(s1, s2, keep = TRUE)
plot(al, type = "twoway")
al$distance
from statsmodels.tsa.arima.model import ARIMA
from statsmodels.tsa.stattools  import adfuller
from prophet  import Prophet
from tslearn.metrics import dtw
import stumpy, numpy as np

# 1) ADF stationarity
adf_stat, p, *_ = adfuller(y)        # p < 0.05 → stationary

# 2) ARIMA(p,d,q) — manual
mod = ARIMA(y, order=(2,1,2)).fit()
fc  = mod.forecast(steps=24)

# 3) Prophet — auto handles trend + seasonality + holidays
df = pd.DataFrame({"ds":dates, "y":y})
m  = Prophet(yearly_seasonality=True, weekly_seasonality=True).fit(df)
future = m.make_future_dataframe(periods=90)
pf  = m.predict(future)

# 4) DTW similarity (variable length)
d = dtw(s1, s2)

# 5) Matrix Profile — motif / discord discovery (STUMPY)
mp = stumpy.stump(y, m=50)              # window 50
motif_idx   = np.argmin(mp[:, 0])         # most-repeating
discord_idx = np.argmax(mp[:, 0])         # most-anomalous

# 6) Change-point detection
import ruptures as rpt
algo = rpt.Pelt(model="rbf").fit(y)
cps  = algo.predict(pen=5)
