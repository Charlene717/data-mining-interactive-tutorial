文字探勘與 NLP 基礎

從詞袋到 BERT——病歷文字、藥品仿單、PubMed 摘要藏著結構化資料無法表達的訊息。

From bag-of-words to BERT — clinical notes, drug labels, and PubMed abstracts carry signals no tabular schema can hold.

概覽

為什麼要做文字探勘？

醫療資料約 80% 為非結構化文字：病歷記錄、影像報告、出院摘要、藥品仿單、文獻。要將這些轉成可學習向量，需經過 (1) 前處理（tokenize → 停用字 → stem/lemma）、(2) 表徵（BoW → TF-IDF → 詞向量 → 句向量）、(3) 建模（分類、主題、摘要、命名實體 NER）。

2018 BERT 之後，NLP 由「特徵工程」進入「預訓練 + 微調」時代。在生醫領域有專門的領域預訓練模型：BioBERT (2019, PubMed/PMC)、PubMedBERT (2020, 全 PubMed scratch)、SciBERT (科學文獻)、BlueBERT (PubMed+MIMIC-III)，常於 BC5CDR、NCBI-disease、ChemProt 等 benchmark 拿到 SOTA。

About 80% of medical data is unstructured text: clinical notes, radiology reports, discharge summaries, drug labels, literature. Turning text into learnable vectors needs (1) preprocessing (tokenize → stopwords → stem/lemma), (2) representation (BoW → TF-IDF → word/sentence vectors), (3) modeling (classification, topics, summarization, NER).

After BERT (2018), NLP shifted from "feature engineering" to "pretrain + fine-tune". Biomedical domain-pretrained models include BioBERT (2019, PubMed/PMC), PubMedBERT (2020, scratch on PubMed), SciBERT (sci. literature), BlueBERT (PubMed+MIMIC-III) — often SOTA on BC5CDR, NCBI-disease, ChemProt.

前處理

一、五大前處理步驟

✂️

① Tokenization

切詞。英文以空白為主；中文/醫學詞需 jieba / ckiptagger / scispaCy。新一代用 BPE / WordPiece (子詞)。

Splitting. English by whitespace; Chinese/medical needs jieba / ckiptagger / scispaCy. Modern models use BPE / WordPiece (sub-word).

🚫

② Stopwords

移除高頻無意義字（the, a, 是, 的）。但醫學情境下保留否定詞 "no", "not"！

Remove high-frequency low-info words (the, a). But in medical text, keep negations like "no", "not"!

🌱

③ Stemming

規則切尾 (Porter, Snowball)：running → run。快但粗糙：「universal → univers」。

Rule-based suffix stripping (Porter, Snowball): running → run. Fast but crude: "universal → univers".

📖

④ Lemmatization

查詞典還原原形：was → be。準但慢；spaCy / WordNetLemmatizer 是常見實作。

Dictionary-based root recovery: was → be. Slower but accurate; spaCy / WordNetLemmatizer are common.

🔢

⑤ Vectorization

BoW (count)、TF-IDF、n-gram、Word2Vec、GloVe、FastText、BERT embedding。

BoW (count), TF-IDF, n-gram, Word2Vec, GloVe, FastText, BERT embedding.

🏷️

⑥ POS / NER

詞性標註 + 命名實體（疾病、藥品、基因）。scispaCy + UMLS linker 為生醫 NER 主流。

POS tagging + named entities (disease, drug, gene). scispaCy + UMLS linker is the biomed NER standard.

💡
TF-IDF 直覺：TF = 詞在當文件出現次數；IDF = log(N / 含該詞的文件數)。在所有文件都出現的詞（the, 病人）IDF≈0 → 整體權重低。在少數文件高頻出現的詞 → 高權重。 TF-IDF intuition: TF = term count in this doc; IDF = log(N / docs containing term). Words in every document (the, patient) have IDF ≈ 0 → low weight. Words frequent in few documents → high weight.
主題模型

三、LDA 主題模型

Latent Dirichlet Allocation (LDA, Blei 2003) 假設：每篇文件是「主題的混合」，每個主題是「字詞的混合」，兩者皆服從 Dirichlet 先驗。產出兩個矩陣：document-topic θ 與 topic-word φ，可同時用於降維、視覺化、主題追蹤。

變體：HDP（自動決定 K）、BTM（Biterm，短文）、CTM / BERTopic（用 BERT embedding + UMAP + HDBSCAN，2022 後常用於 PubMed/COVID 文獻洞察）。

Latent Dirichlet Allocation (LDA, Blei 2003) assumes each document is a mixture of topics and each topic is a mixture of words, with Dirichlet priors. It yields two matrices: document-topic θ and topic-word φ — used for dim-reduction, visualization, topic tracking.

Variants: HDP (auto-K), BTM (short text), CTM / BERTopic (BERT embedding + UMAP + HDBSCAN, popular for PubMed/COVID literature mining post-2022).

程式碼

實作：TF-IDF → Word2Vec → LDA → BioBERT

library(tm); library(tidytext); library(text2vec); library(topicmodels)

# 1) TF-IDF（tidytext）
docs <- tibble(id = 1:3, text = c("patient with sepsis fever",
                                  "diabetes glucose insulin patient",
                                  "sepsis shock icu admission"))
tf_idf <- docs |>
  unnest_tokens(word, text) |>
  count(id, word) |>
  bind_tf_idf(word, id, n)

# 2) Word2Vec（text2vec, GloVe-style）
it    <- itoken(tolower(docs$text), tokenizer = word_tokenizer)
vocab <- create_vocabulary(it)
v_pr  <- prune_vocabulary(vocab, term_count_min = 2)
tcm   <- create_tcm(it, vocab_vectorizer(v_pr), skip_grams_window = 5)
glove <- GlobalVectors$new(rank = 50, x_max = 10)
wv    <- glove$fit_transform(tcm, n_iter = 10)

# 3) LDA 主題模型
dtm <- cast_dtm(count(unnest_tokens(docs, word, text), id, word), id, word, n)
lda <- topicmodels::LDA(dtm, k = 2, control = list(seed = 42))
tidy(lda, matrix = "beta")         # topic-word
from sklearn.feature_extraction.text import TfidfVectorizer, CountVectorizer
from gensim.models import Word2Vec, LdaModel
from gensim.corpora import Dictionary
from transformers   import AutoTokenizer, AutoModel
import torch

docs = ["patient with sepsis fever",
        "diabetes glucose insulin patient",
        "sepsis shock icu admission"]

# 1) TF-IDF
tfidf = TfidfVectorizer(stop_words="english", ngram_range=(1,2))
X = tfidf.fit_transform(docs)        # sparse matrix

# 2) Word2Vec（Skip-gram, sg=1; CBOW, sg=0）
sents = [d.split() for d in docs]
w2v = Word2Vec(sents, vector_size=100, window=5,
               min_count=1, sg=1, epochs=50)
vec_sepsis = w2v.wv["sepsis"]

# 3) LDA
dct  = Dictionary(sents)
bow  = [dct.doc2bow(s) for s in sents]
lda  = LdaModel(corpus=bow, id2word=dct, num_topics=2, passes=10)
print(lda.show_topics())

# 4) BioBERT 句向量（biomedical）
tok = AutoTokenizer.from_pretrained("dmis-lab/biobert-v1.1")
mod = AutoModel.from_pretrained("dmis-lab/biobert-v1.1")
inp = tok(docs, padding=True, return_tensors="pt")
with torch.no_grad():
    emb = mod(**inp).last_hidden_state.mean(dim=1)   # [N,768]
⚠️
領域偏移：用 BERT (general) 處理 PubMed 表現往往遜於 BioBERT/PubMedBERT。在病歷 (clinical) 還需用 ClinicalBERT 或 Bio_ClinicalBERT (Alsentzer 2019)。「英文 ≠ 醫學英文」。 Domain shift: BERT (general) on PubMed often loses to BioBERT/PubMedBERT. For clinical notes you also need ClinicalBERT or Bio_ClinicalBERT (Alsentzer 2019). "English ≠ medical English".
