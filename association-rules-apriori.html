關聯規則探勘

「買啤酒的人也常買尿布」——關聯規則的故事人盡皆知，但背後的支援度、信賴度、Lift 三個基本量才是評估規則的真功夫。

"People who buy beer also buy diapers." The story is famous; the real craft is the trio of support, confidence, and lift that judges whether a rule is meaningful.

概覽

交易、項集、規則

輸入資料是「交易資料庫」：每筆交易 T 是一個項目集合（如購物車、共現症狀、共處方）。任意項目組合稱為 itemset，被夠多交易支持的稱為 frequent itemset。形如 X ⇒ Y 的稱為 關聯規則（X、Y 互不相交）。

生物醫學應用：共現症狀網絡、共處方藥物模式、SNP 共現、ICD 碼共現、單細胞共表達。Nature Sci Rep 2026 提出的 ASEA（Association Subgraph Enrichment Analysis）就是把關聯規則與基因本體 (Gene Ontology) 結合的 ontology-driven 變體。

The input is a transaction database: each transaction T is a set of items (shopping cart, co-occurring symptoms, co-prescriptions). Any item combination is an itemset; if enough transactions support it, it's frequent. An association rule takes the form X ⇒ Y (X and Y disjoint).

Biomedical uses: symptom co-occurrence networks, co-prescription patterns, SNP co-occurrence, ICD code combinations, single-cell co-expression. ASEA (Association Subgraph Enrichment Analysis, Nature Sci Rep 2026) is an ontology-driven variant that fuses association rules with the Gene Ontology.

三大指標

一、Support / Confidence / Lift

指標公式意義
SupportP(X∪Y) = |T含X∪Y| / |T|該組合出現的頻率，過低代表罕見Frequency of the combination — low = rare
ConfidenceP(Y|X) = supp(X∪Y) / supp(X)在含 X 的交易中，含 Y 的比例Among X-transactions, fraction also containing Y
LiftP(Y|X) / P(Y) = conf / supp(Y)>1 正相關、=1 獨立、<1 負相關>1 positive, =1 independent, <1 negative
LeverageP(X∪Y) − P(X)·P(Y)超出獨立假設的「絕對」差Absolute deviation from independence
Conviction(1−supp(Y)) / (1−conf)∞ 表示 X 完全暗示 Y；1 表示獨立∞ if X always implies Y; 1 if independent
💡
為什麼需要 Lift？高信賴度可能只是因為 Y 本來就常見（如「啤酒 ⇒ 麵包」conf=0.9，但其實買麵包的本來就 0.92）。Lift > 1 才代表 X 真的「提升」了 Y 出現的可能性。 Why lift? High confidence can simply reflect Y being common (e.g., beer ⇒ bread with 0.9 confidence, but bread itself is bought by 92%). Lift > 1 means X genuinely raises the chance of Y.
演算法

二、三大演算法

Apriori (1994)

逐層搜尋：先找 1-itemset，再合併成候選 2-itemset，掃資料庫過濾，再 3-itemset…。核心利器是 反單調性 (anti-monotone)：若任一子集不頻繁，超集必不頻繁——可大幅剪枝。

Level-wise search: find 1-itemsets, join into candidate 2-itemsets, scan DB to prune, then 3-itemsets... Key: the anti-monotone property — if a subset is infrequent, every superset must be — enabling massive pruning.

FP-Growth (2000)

建立壓縮的 FP-tree，僅需兩次掃資料庫，避免 Apriori 的候選集爆炸。在密集資料上比 Apriori 快 10–100×。

Builds a compact FP-tree; only two DB scans, avoiding Apriori's candidate explosion. On dense data, 10–100× faster than Apriori.

Eclat (2000)

使用 垂直 tidlist（每個 item 記錄它出現在哪些交易 ID）。藉由 tidlist 交集計算支援度，記憶體效率高，適合中等大小資料。

Uses vertical tidlists (each item lists transaction IDs it appears in). Support is computed by tidlist intersection — memory-efficient, good for mid-sized data.

⚠️
規則數爆炸：低 support 可產生數百萬規則。實務上會 (1) 設定足夠的 min_support，(2) 取 Top-N by Lift，(3) 用 closed 或 maximal itemset 限縮輸出，(4) 視覺化用 arulesViz 的 grouped matrix / graph。 Rule explosion: low support yields millions of rules. In practice: (1) set min_support high enough, (2) take Top-N by lift, (3) restrict to closed or maximal itemsets, (4) visualize with arulesViz grouped matrix / graph.

決策樹

三、演算法與規則篩選

🌳 關聯規則探勘決策樹

Q1:
小資料、教學示範、需逐層解釋？ → Apriori (arules / mlxtend)
Q2:
大型密集資料？ → FP-Growth（更少 DB 掃描）
Q3:
中型稀疏資料、記憶體有限？ → Eclat（垂直 tidlist 交集）
Q4:
規則太多想壓縮？ → 只保留 closed 或 maximal itemset
Q5:
需驗證統計顯著性？ → Fisher exact + BH 多重檢定校正
Q6:
生醫資料需結合本體？ → ASEA（Gene Ontology + 關聯規則）
Q1:
Small data, teaching, need step-by-step? → Apriori (arules / mlxtend)
Q2:
Large dense data? → FP-Growth (fewer DB scans)
Q3:
Mid-size sparse data, memory-tight? → Eclat (vertical tidlist intersect)
Q4:
Too many rules — compress? → Keep closed or maximal itemsets only
Q5:
Need statistical significance? → Fisher exact + BH multiple-testing correction
Q6:
Biomedical data with ontology context? → ASEA (Gene Ontology + rules)

from mlxtend.preprocessing import TransactionEncoder
from mlxtend.frequent_patterns import apriori, fpgrowth, association_rules

# 4) 產生並按 lift 排序 / generate rules sorted by lift
rules = association_rules(fi_fp, metric="lift", min_threshold=1.2)
rules = rules[(rules["confidence"] >= 0.30) &

        .sort_values("lift", ascending=False)
print(rules[["antecedents","consequents","support","confidence","lift"]].head(10))
