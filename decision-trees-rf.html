CHAPTER 6 / 20 · MODULE B

決策樹、隨機森林與梯度提升

樹型模型是表格資料的霸主——從可解釋的單棵 CART，到實務 KPI 標配的 XGBoost、LightGBM、CatBoost。本章帶你從分裂準則一路走到 2025 年最新提升器版本。

Tree-based models reign on tabular data — from interpretable single CART to industry-standard XGBoost, LightGBM, CatBoost. We walk from splitting criteria to the 2025 generation of boosters.

概覽

為什麼樹模型在表格資料上稱王？

樹模型 (decision trees) 透過遞迴二元切分讓資料逐步「純化」，天然處理非線性、特徵交互、不同單位、缺失值與類別變數，且無需 scaling。集成版本 (Random Forest、Gradient Boosting) 進一步穩定變異與抑制偏差——在 Kaggle 結構化資料賽事中，2015 之後幾乎都是 XGBoost / LightGBM / CatBoost 拿冠軍。

但單棵樹對小變動極敏感（high variance），因此實務上鮮少獨自使用；它的價值在於可解釋的規則（臨床決策、合規場景）和集成的基底單元。

Decision trees recursively split data into purer subsets — natively handling nonlinearity, feature interactions, mixed units, missing values, and categoricals, all without scaling. Ensembles (Random Forest, Gradient Boosting) tame variance and bias further — since 2015, nearly every Kaggle tabular winner has used XGBoost / LightGBM / CatBoost.

A single tree is extremely sensitive to small data changes (high variance); its real value is as an interpretable rule set (clinical decisions, compliance) and as the base learner for ensembles.

過擬合

二、樹的深度 vs 過擬合

單棵樹若不剪枝會長到每個葉子只含一筆資料——訓練準確率 100%、驗證準確率崩壞。控制方法：max_depth、min_samples_split、min_samples_leaf、ccp_alpha (cost-complexity pruning)。拖動下方深度滑桿觀察 train/val 曲線分離。

集成方法

三、集成方法

🌲

Random Forest

Bagging（bootstrap aggregating）+ 每次分裂只看隨機 √p 個特徵 → 降低樹間相關性，顯著減少變異。平行可訓練。

Bagging (bootstrap aggregating) + random √p feature subset at each split → decorrelates trees, greatly reduces variance. Embarrassingly parallel.

🚀

Gradient Boosting

每棵新樹擬合前一輪的殘差 / 負梯度，序列建構。降低偏差為主，常勝過 RF 但易過擬合。

Each new tree fits the previous round's residuals / negative gradient. Sequential. Lowers bias mostly; often beats RF but overfits more easily.

🏆

決策樹

四、什麼時候用哪一個？

🌳 樹型模型選擇決策樹

Q1:
需要單一可解釋規則（臨床指引、稽核）？ → 單棵 CART (rpart / DecisionTree)
Q2:
追求穩健 baseline、容易調？ → Random Forest
Q3:
追求最高準確率且接受長一點的調參？ → XGBoost / LightGBM
Q4:
資料含大量高基數類別（地區、商品、ICD code）？ → CatBoost
Q5:
資料超大（>10M rows）？ → LightGBM（leaf-wise + Goss）
Q1:
Need a single interpretable rule (clinical guideline, audit)? → Single CART (rpart / DecisionTree)
Q2:
Want a robust, easy-to-tune baseline? → Random Forest
Q3:
Need the highest accuracy and willing to tune? → XGBoost / LightGBM
Q4:
Many high-cardinality categoricals (region, SKU, ICD code)? → CatBoost
Q5:
Huge data (>10M rows)? → LightGBM (leaf-wise + Goss)
⚠️
RF vs GBM：RF 平行訓練、抗過擬合、易調 → 適合 baseline；GBM 通常更準但需要早停 (early_stopping_rounds) 與較多 hyperparam tuning。 RF vs GBM: RF is parallel, robust, easy to tune — great baseline. GBM is usually more accurate but needs early stopping and heavier tuning.

# 2. Random Forest (ranger — 快) / Random Forest

bst <- xgboost::xgb.train(params=list(objective="multi:softprob", num_class=3,
                eta=0.1, max_depth=4, eval_metric="mlogloss"),
                data=dtrain, nrounds=200, early_stopping_rounds=20,
                watchlist=list(train=dtrain))

# 4. LightGBM 4.x
lgb_data <- lightgbm::lgb.Dataset(as.matrix(iris[,1:4]), label=as.integer(iris$Species)-1)

                num_leaves=15), data=lgb_data, nrounds=200)
