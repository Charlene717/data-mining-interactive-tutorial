PubMed 與文獻探勘

3,800 萬篇摘要、每年新增百萬——把生醫文獻變成可查詢、可推理的知識圖譜。

38M abstracts, growing by ~1M / year — turning biomedical literature into a queryable, inferable knowledge graph.

概覽

從關鍵字查詢到知識圖譜

PubMed 由 NLM 維護，2025 年已超過 3,800 萬筆引用。文獻探勘的層次大致為：(1) 檢索 / Boolean，(2) 實體辨識 (NER)，(3) 關係抽取 (RE)，(4) 知識圖譜 (KG) 建構與推理，(5) 藥物再利用 (drug repurposing) 與 假說生成 (Swanson ABC)。

近年 LLM/領域 BERT 大幅提升了 NER 與 RE 的 F1：BioBERT (2019)、PubMedBERT (2020)、SciSpacy、BERN2 (2022)、BioGPT (2022)、Med-Gemini (2024)、BioMistral 7B (2024)。但幻覺 (hallucination) 與引用偽造仍是 LLM 在文獻萃取上的最大風險。

PubMed (NLM) holds 38M+ citations as of 2025. Literature mining layers roughly as: (1) retrieval / Boolean, (2) NER, (3) relation extraction, (4) knowledge-graph construction & inference, (5) drug repurposing and hypothesis generation (Swanson's ABC).

Domain BERTs and LLMs lift NER/RE F1 dramatically: BioBERT (2019), PubMedBERT (2020), SciSpacy, BERN2 (2022), BioGPT (2022), Med-Gemini (2024), BioMistral 7B (2024). But hallucination and fabricated citations remain the biggest LLM risks in literature extraction.

NER 與 RE

二、命名實體辨識 vs 關係抽取

🏷️

NER

把 token 分類為 GENE / DISEASE / DRUG / CHEMICAL / SPECIES。代表：SciSpacy（en_ner_bionlp13cg_md）、PubTator Central、BERN2 (2022)、BioMedical-NER-All。

Classify tokens as GENE / DISEASE / DRUG / CHEMICAL / SPECIES. Tools: SciSpacy (en_ner_bionlp13cg_md), PubTator Central, BERN2 (2022), BioMedical-NER-All.

↔️

RE

判斷兩實體間關係：treats / causes / inhibits / interacts_with。代表：BioRED (2022)、ChemProt、DDI corpus、GDA / CDR (BioCreative)。

Classify the relation between two entities: treats / causes / inhibits / interacts_with. Datasets: BioRED (2022), ChemProt, DDI corpus, GDA / CDR (BioCreative).

藥物再利用 & KG

三、知識圖譜與藥物再利用

主流生醫 KG：Hetionet (47k node / 2.25M edge, 2017)、BioKG、DRKG (COVID-19, 2020)、PrimeKG (Harvard, 2023/2024 — 4M edges, 17 sources)、OpenTargets Platform、DrugBank 5。藥物再利用核心：在 (drug)–[?]→(disease) 邊上做 link prediction（如 RotatE、TransE、ComplEx、HGNN）。著名案例：sildenafil 從 angina → erectile dysfunction → pulmonary hypertension。

Swanson 的 ABC 假說：若 A→B 與 B→C 都在文獻中，但 A→C 未被連起來，就是可能的新假說（1986 經典 Raynaud × fish oil）。

Mainstream biomedical KGs: Hetionet (47k nodes / 2.25M edges, 2017), BioKG, DRKG (COVID-19, 2020), PrimeKG (Harvard, 2023/2024 — 4M edges, 17 sources), OpenTargets Platform, DrugBank 5. Drug repurposing core: link prediction on (drug)–[?]→(disease) edges (RotatE, TransE, ComplEx, HGNN). Classic: sildenafil from angina → ED → pulmonary hypertension.

Swanson's ABC hypothesis: if A→B and B→C exist in literature but A→C is not yet linked, that is a candidate new hypothesis (1986: Raynaud × fish oil).

程式碼

實作：E-utilities + SciSpacy NER + KG

library(easyPubMed); library(pubmed.mineR); library(udpipe); library(igraph)

# --- query PubMed via E-utilities ---
q       <- 'metformin[MeSH] AND "Type 2 Diabetes"[MeSH] AND 2024[PDAT]'
ids     <- get_pubmed_ids(q)
abs_xml <- fetch_pubmed_data(ids, retmax = 500)
df      <- articles_to_list(abs_xml) |> articles_to_dataframe()

# --- udpipe biomedical model + tokenize + lemmatize ---
m  <- udpipe::udpipe_download_model(language = "english-ewt")
ud <- udpipe::udpipe_load_model(m$file_model)
tk <- udpipe::udpipe_annotate(ud, x = df$abstract) |> as.data.frame()

# --- co-occurrence at sentence level & KG ---
co <- udpipe::cooccurrence(tk, term = "lemma", group = c("doc_id","sentence_id"))
g  <- igraph::graph_from_data_frame(co)
igraph::plot(g, vertex.size = 5, edge.arrow.size = 0.3)
from Bio import Entrez
import scispacy, spacy
from spacy.lang.en import English
import networkx as nx
Entrez.email = "you@uni.edu"; Entrez.api_key = "…"

# --- E-utilities: ESearch + EFetch ---
h    = Entrez.esearch(db="pubmed", term='metformin[MeSH] AND "Type 2 Diabetes"[MeSH] AND 2024[PDAT]', retmax=500)
ids  = Entrez.read(h)["IdList"]
abs_ = Entrez.efetch(db="pubmed", id=ids, rettype="abstract", retmode="xml").read()

# --- SciSpacy NER (biomed) + linker to UMLS ---
nlp = spacy.load("en_ner_bionlp13cg_md")
from scispacy.linking import EntityLinker
nlp.add_pipe("scispacy_linker", config={"linker_name": "umls", "resolve_abbreviations": True})

# --- entity extraction + sentence co-occurrence KG ---
G = nx.Graph()
for doc in nlp.pipe(abstracts, batch_size=32):
    for sent in doc.sents:
        ents = [(e.text, e.label_) for e in sent.ents]
        for a, b in combinations(ents, 2):
            G.add_edge(a[0], b[0], rel=f"{a[1]}-{b[1]}")

# --- LLM-augmented RE with PubMedBERT ---
from transformers import pipeline
re = pipeline("text-classification", model="microsoft/BiomedNLP-PubMedBERT-base-uncased-abstract-fulltext")
