相似度與距離度量

幾乎所有探勘演算法（KNN、K-Means、Hierarchical、UMAP、DBSCAN）都依賴一個核心問題：「兩筆資料有多像？」

Nearly every mining algorithm (KNN, K-Means, Hierarchical, UMAP, DBSCAN) hinges on one question: "How similar are two records?"

概覽

距離度量的四個性質

嚴格意義的「距離度量 (metric)」必須滿足四個公理：(1) 非負 d(x,y)≥0、(2) 同一性 d(x,y)=0 ⇔ x=y、(3) 對稱 d(x,y)=d(y,x)、(4) 三角不等式 d(x,z) ≤ d(x,y)+d(y,z)。不滿足全部四項的稱為 半度量 或 相似度（如 Cosine 不是嚴格距離）。

距離選擇直接決定模型行為——KNN 預測、K-Means cluster 形狀、Hierarchical 樹結構都會因此改變。在生物醫學上更要小心：基因表現量用 Cosine / Pearson，影像 fingerprint 用 Tanimoto / Jaccard，混合型病人特徵用 Gower，ECG 時間序列用 DTW。

A strict metric satisfies four axioms: (1) non-negativity d(x,y)≥0, (2) identity d(x,y)=0 ⇔ x=y, (3) symmetry d(x,y)=d(y,x), (4) triangle inequality d(x,z) ≤ d(x,y)+d(y,z). Failing any of these makes it a semi-metric or just a similarity (e.g., Cosine is not a strict metric).

The choice of distance shapes model behavior — KNN predictions, K-Means cluster shapes, hierarchical trees all change. In biomedicine: gene expression uses Cosine / Pearson, chemical fingerprints use Tanimoto / Jaccard, mixed patient features use Gower, ECG time-series uses DTW.

決策樹

二、如何挑選正確的距離？

🌳 距離選擇決策樹

Q1:
資料是純二元/集合（symptoms, SNPs, fingerprints）？ → Jaccard / Tanimoto / Dice
Q2:
資料是連續且方向比尺度重要（gene expression、TF-IDF）？ → Cosine / Pearson
Q3:
資料是連續同單位低維？ → Euclidean（記得先 scaling）
Q4:
資料是高維且雜訊多？ → Manhattan / Cosine（避免「維度詛咒」）
Q5:
資料是混合型 EHR（age + sex + 用藥）？ → Gower (R: cluster::daisy)
Q6:
資料是時間序列且長度不同？ → DTW (dtaidistance / tslearn)
Q7:
需考慮特徵間共變異？ → Mahalanobis
Q1:
Pure binary / set data (symptoms, SNPs, fingerprints)? → Jaccard / Tanimoto / Dice
Q2:
Continuous, direction matters more than magnitude (gene expression, TF-IDF)? → Cosine / Pearson
Q3:
Continuous, same units, low-D? → Euclidean (after scaling!)
Q4:
High-D and noisy? → Manhattan / Cosine (mitigate curse of dimensionality)
Q5:
Mixed EHR (age + sex + meds)? → Gower (R: cluster::daisy)
Q6:
Time series of differing length? → DTW (dtaidistance / tslearn)
Q7:
Need covariance awareness? → Mahalanobis
⚠️
維度詛咒：在高維空間中（>30 維），所有點之間的 Euclidean 距離趨於相等，使 KNN 變得無意義。解法：先 PCA / UMAP 降維，或改用 Cosine / Manhattan。 Curse of dimensionality: In high-D (>30), all pairwise Euclidean distances converge — KNN becomes meaningless. Fix: PCA / UMAP first, or use Cosine / Manhattan.
