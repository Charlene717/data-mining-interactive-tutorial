CHAPTER 10 / 20 · MODULE C

密度型與機率型分群

當群非球狀、密度不均、有雜訊，K-Means 就失效。本章介紹 DBSCAN、OPTICS、HDBSCAN、Mean-Shift、GMM、Spectral——形狀無偏見的分群家族。

When clusters are non-spherical, vary in density, or contain noise, K-Means breaks down. This chapter covers DBSCAN, OPTICS, HDBSCAN, Mean-Shift, GMM, and Spectral — the shape-agnostic clustering family.

概覽

為什麼需要密度型分群？

K-Means 三大限制：(1) 必須事先指定 k，(2) 假設群為球狀，(3) 把每個點都強制指派到群（包含 outlier）。在許多生物醫學情境裡，群數未知、形狀為彎月或環、且含明顯離群——這時 密度型 (density-based) 方法閃耀登場。

密度型方法的核心觀念：群 = 高密度區域的連通元件，noise = 低密度區域的孤立點。代表算法為 1996 年 KDD 上提出的 DBSCAN，至今仍是 sklearn / R 的標準工具。

K-Means has three big limits: (1) k must be set, (2) clusters assumed spherical, (3) every point is forced into a cluster (including outliers). In many biomedical settings clusters are crescent- or ring-shaped, k is unknown, and there are real outliers — this is where density-based methods shine.

Core idea: clusters are connected components of high density, while noise sits in low-density regions. The flagship algorithm is DBSCAN (KDD 1996), still the standard in sklearn / R.

DBSCAN

一、DBSCAN：三類點與兩個參數

Core 核心點

在半徑 ε 內至少有 minPts 個鄰居。是群的「種子」。

Has at least minPts neighbors within radius ε. The cluster seed.

Border 邊界點

鄰居數 < minPts，但落在某 core 的 ε 內。被吸入該群但不再擴張。

Has < minPts neighbors, but lies within ε of some core. Absorbed but does not extend further.

Noise 雜訊點

既不是 core 也不在任何 core 的 ε 鄰域內。標記為 -1（不屬於任何群）。

Neither core nor within any core's ε. Labelled -1 (no cluster).

💡
選 ε 的經驗法則：畫 k-distance plot（k = minPts，遞減排序），找肘部。選 minPts：通常取 2×維度（如 2D 用 minPts=4）。 Choosing ε: plot the k-distance graph (k = minPts, sorted descending), find the knee. minPts: rule of thumb is 2 × dim (4 for 2D).

家族比較

二、密度型 / 機率型分群家族

演算法關鍵概念優勢
DBSCANε, minPts無需 k；可偵測雜訊；任意形狀No k; detects noise; arbitrary shape
OPTICSreachability plot不需固定 ε，可擷取多層次群No fixed ε; multi-level clusters
HDBSCANcondensed tree處理變化密度；提供成員資格機率Handles varying density; soft membership
Mean-Shift爬山找密度峰無需 k；自動找模式No k; finds modes automatically
GMM (EM)k 個高斯的混合軟分群；BIC 選 k；橢圓形群Soft assignment; BIC for k; elliptical
Spectral圖 Laplacian非凸群；流形結構Non-convex clusters; manifold-aware
⚠️
HDBSCAN 改進 DBSCAN 的關鍵：引入 mutual reachability distance 與 condensed cluster tree，使其能處理「密度不均」的群——例如同時存在 1000 個與 30 個成員的群，DBSCAN 必須妥協單一 ε，HDBSCAN 不必。 HDBSCAN improves DBSCAN by: using mutual reachability distance and a condensed cluster tree, letting it handle varying-density clusters — e.g. one with 1000 members and another with 30 — without a single global ε.
GMM 與 EM

三、GMM 與軟分群

GMM 假設資料來自 k 個多元高斯的混合：p(x)=Σ π_k · 𝒩(x | μ_k, Σ_k)。以 EM 演算法交替更新：(E-step) 算每點屬於每群的機率 (responsibility)；(M-step) 用 responsibility-weighted 更新 π, μ, Σ。

k 的選擇用 BIC（越小越好，內建懲罰複雜度）。共變異形式有 full（橢圓任意角度）、diagonal（橢圓但軸對齊）、spherical（圓形——退化為 K-Means）、tied（共用 Σ）。

GMM models data as a mixture of k Gaussians: p(x) = Σ π_k · 𝒩(x | μ_k, Σ_k). The EM algorithm alternates: (E-step) compute each point's responsibility for each component; (M-step) update π, μ, Σ using those weights.

Pick k by BIC (lower is better, penalizes complexity). Covariance types: full (any-angle ellipses), diagonal (axis-aligned ellipses), spherical (circles — reduces to K-Means), tied (shared Σ).

決策樹

四、方法選擇決策樹

🌳 密度型分群決策樹

Q1:
群形狀奇怪、有雜訊、密度均勻？ → DBSCAN
Q2:
群密度差異大（有稀疏 + 緊密混合）？ → HDBSCAN
Q3:
想看多解析度結構或不確定 ε？ → OPTICS
Q4:
群為橢圓且需「軟分群機率」？ → GMM (full / diagonal Σ)
Q5:
群為非凸流形（環、彎月、瑞士卷）？ → Spectral Clustering
Q6:
不知道 k 也不想調 ε，只想自動找密度峰？ → Mean-Shift（小資料）
Q1:
Irregular shape, noise, uniform density? → DBSCAN
Q2:
Varying density (sparse + tight mixed)? → HDBSCAN
Q3:
Want multi-resolution view / ε unclear? → OPTICS
Q4:
Elliptical clusters and want soft probabilities? → GMM (full / diagonal Σ)
Q5:
Non-convex manifolds (rings, crescents, swiss roll)? → Spectral Clustering
Q6:
No k, no ε tuning, just find density modes? → Mean-Shift (small N)

實作

library(dbscan); library(mclust); library(kernlab)
X <- scale(iris[, 1:4])

# 1) DBSCAN — 先看 k-distance plot 決定 eps
dbscan::kNNdistplot(X, k = 4); abline(h = 0.5, lty = 2)
db <- dbscan::dbscan(X, eps = 0.5, minPts = 4)
table(db$cluster)   # 0 = noise / 0 = noise

# 2) OPTICS — 不需固定 eps / no fixed eps
op <- dbscan::optics(X, minPts = 4)
plot(op)            # reachability plot

# 3) HDBSCAN — 處理密度變化 / varying density
hdb <- dbscan::hdbscan(X, minPts = 5)

# 4) GMM with BIC 自動選 k / GMM with BIC auto-k
gmm <- mclust::Mclust(X, G = 1:9)
gmm$G; gmm$BIC

# 5) Spectral Clustering
sc <- kernlab::specc(X, centers = 3)
from sklearn.cluster import DBSCAN, OPTICS, SpectralClustering, MeanShift
from sklearn.mixture import GaussianMixture
from sklearn.neighbors import NearestNeighbors
import hdbscan, numpy as np

# 1) DBSCAN — k-distance heuristic for eps
nn = NearestNeighbors(n_neighbors=4).fit(X)
d, _ = nn.kneighbors(X); d = np.sort(d[:, -1])
# 肘部就是建議 eps / knee gives suggested eps
db = DBSCAN(eps=0.5, min_samples=4).fit(X)
print(np.unique(db.labels_, return_counts=True))    # -1 = noise

# 2) OPTICS
op = OPTICS(min_samples=5, xi=0.05).fit(X)

# 3) HDBSCAN — 推薦現代標準 / modern default
hdb = hdbscan.HDBSCAN(min_cluster_size=15, min_samples=5).fit(X)
hdb.probabilities_   # 軟成員機率 / soft membership probability

# 4) GMM + BIC 自動選 k
bics = []
for k in range(1, 10):
    m = GaussianMixture(n_components=k, covariance_type="full").fit(X)
    bics.append((k, m.bic(X)))
best_k = min(bics, key=lambda kb: kb[1])[0]

# 5) Spectral Clustering for non-convex shapes / 非凸群
sc = SpectralClustering(n_clusters=3, affinity="nearest_neighbors").fit(X)
