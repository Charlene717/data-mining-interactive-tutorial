K-Means 與階層式分群

分群是探索性學習的代表：給未標記的資料找到「自然的群組」。K-Means 重速度、Hierarchical 重結構，兩者互補。

Clustering is the flagship of exploratory learning: find natural groups in unlabeled data. K-Means is fast; hierarchical reveals structure — they complement each other.

概覽

兩大派典：劃分式 vs 階層式

劃分式 (partitional)：使用者事先指定 k，演算法找到使類內平方和 (WCSS) 最小的指派。代表：K-Means、K-Medoids (PAM)、CLARA。

階層式 (hierarchical)：建立樹狀 dendrogram，可由下而上 (agglomerative) 或由上而下 (divisive)。不需事先決定 k——切樹高度即得分群。

在生物醫學應用上：基因表現的 patient subtyping（TCGA 各 cancer type 都用 Ward + Euclidean 起家），單細胞 RNA-seq 用 Leiden / Louvain，臨床表型分群常用 K-Medoids + Gower。

Partitional: user specifies k, the algorithm finds an assignment that minimizes within-cluster sum of squares (WCSS). Examples: K-Means, K-Medoids (PAM), CLARA.

Hierarchical: builds a dendrogram, either bottom-up (agglomerative) or top-down (divisive). k need not be pre-specified — cut the tree at a chosen height.

Biomedical examples: patient subtyping from gene expression (TCGA tumor-type analyses often start with Ward + Euclidean), single-cell RNA-seq uses Leiden / Louvain, and clinical phenotype clustering often pairs K-Medoids with Gower distance.

K-MEANS 細節

一、Lloyd 演算法與 k-means++

Lloyd 迭代

1) 初始化 k 個質心。2) 指派每點到最近質心。3) 重算每群平均成為新質心。4) 重複至收斂或 max_iter。

1) Initialize k centroids. 2) Assign each point to nearest centroid. 3) Recompute means as new centroids. 4) Repeat until convergence or max_iter.

k-means++

隨機初始化常陷入區域最小值。k-means++ 以「距離平方加權」依序挑質心，提高初始品質——sklearn 與 R 的 stats::kmeans 預設皆採用。

Random init often stalls at local minima. k-means++ picks subsequent seeds with probability proportional to squared distance — both sklearn and R's stats::kmeans use it by default.

⚠️
K-Means 的隱含假設：(1) 群為球狀且等變異、(2) 群大小相近、(3) 特徵須事先 scaling。違反任一條件時，DBSCAN / GMM 更合適。 K-Means assumes: (1) clusters are spherical with equal variance, (2) similar cluster sizes, (3) features are pre-scaled. Violate any of these and DBSCAN / GMM may be better.

三、Linkage 比較與 dendrogram 切法

Linkage定義形狀偏好
Singlemin d(x,y), x∈A, y∈B鏈狀 (chaining)，對 outlier 敏感Chaining; outlier-sensitive
Completemax d(x,y)緊湊球狀，怕內部噪聲Compact spheres; noise-averse
Average (UPGMA)兩點對平均折衷；演化樹常用Balanced; common in phylogenetics
Ward最小化 WCSS 增量球狀、大小相近；最常用Spherical, equal-size; most common
💡
Cophenetic correlation：dendrogram 中兩點被合併時的高度 vs 原始距離的相關係數。>0.75 通常被認為 dendrogram 忠實反映原距離結構。 Cophenetic correlation: correlation between dendrogram merge heights and original pairwise distances. >0.75 usually means the tree faithfully preserves the distance structure.
決策樹

四、K-Means 還是 Hierarchical？

🌳 分群演算法決策樹

Q1:
資料量 > 10 萬筆 且 已知 k？ → K-Means / MiniBatchKMeans
Q2:
群為球狀、特徵已 scale？ → K-Means (k-means++ 初始化)
Q3:
有 outlier 或希望中位數型代表？ → K-Medoids (PAM)
Q4:
需可解釋的樹狀結構 / 不確定 k？ → Hierarchical (Ward + Euclidean)
Q5:
群形狀奇怪、密度不均？ → DBSCAN / HDBSCAN (Ch 10)
Q1:
>100k rows and k known? → K-Means / MiniBatchKMeans
Q2:
Spherical clusters, scaled features? → K-Means (k-means++ init)
Q3:
Outliers present or want medoid representatives? → K-Medoids (PAM)
Q4:
Need interpretable tree / unsure of k? → Hierarchical (Ward + Euclidean)
Q5:
Irregular shape, varying density? → DBSCAN / HDBSCAN (Ch 10)

library(cluster); library(factoextra)
X <- scale(iris[, 1:4])

# 1) K-Means（k-means++ 由 nstart 模擬）/ K-Means with multi-start
km <- stats::kmeans(X, centers = 3, nstart = 25, iter.max = 100)
km$tot.withinss        # WCSS
factoextra::fviz_nbclust(X, kmeans, method = "wss")         # Elbow
factoextra::fviz_nbclust(X, kmeans, method = "silhouette") # Silhouette
factoextra::fviz_nbclust(X, kmeans, method = "gap_stat", nboot = 50)

# 2) K-Medoids (PAM) — outlier-robust
pam.res <- cluster::pam(X, k = 3, metric = "euclidean")

# 3) Hierarchical + Ward / Hierarchical with Ward linkage
d  <- dist(X, method = "euclidean")
hc <- stats::hclust(d, method = "ward.D2")
clu <- cutree(hc, k = 3)
coph <- cor(d, cophenetic(hc))     # >0.75 = 忠實
factoextra::fviz_dend(hc, k = 3, rect = TRUE)
from sklearn.cluster import KMeans, AgglomerativeClustering
from sklearn.metrics import silhouette_score
from sklearn.preprocessing import StandardScaler
from scipy.cluster.hierarchy import linkage, dendrogram, fcluster, cophenet
from scipy.spatial.distance import pdist
import numpy as np

X = StandardScaler().fit_transform(X_raw)

# 1) K-Means with k-means++ / K-Means 內建 k-means++
km = KMeans(n_clusters=3, init="k-means++", n_init=25, random_state=42).fit(X)
print(km.inertia_, silhouette_score(X, km.labels_))

# Elbow + silhouette sweep / 一次掃過多個 k
for k in range(2, 8):
    m = KMeans(n_clusters=k, n_init=10).fit(X)
    print(k, m.inertia_, silhouette_score(X, m.labels_))

# 2) Hierarchical (Ward) + dendrogram cut
Z = linkage(X, method="ward")
labels = fcluster(Z, t=3, criterion="maxclust")
coph_corr, _ = cophenet(Z, pdist(X))   # 0.75+ 良好

# 3) sklearn AgglomerativeClustering / sklearn 版本
agg = AgglomerativeClustering(n_clusters=3, linkage="ward").fit(X)
