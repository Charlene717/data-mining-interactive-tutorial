組學資料探勘

當 p ≫ n：在數萬個基因 / 蛋白 / 代謝物中找出真正有意義的少數，並把多個 -omics 層次整合。

When p ≫ n: pinpointing the meaningful few among tens of thousands of genes / proteins / metabolites, and integrating across omics layers.

概覽

高維小樣本問題 (p ≫ n)

典型轉錄組研究：20,000 個基因 × 50 個樣本，p/n = 400。傳統 OLS 無解、Ridge 過擬合、單變項 t-test 校正完幾乎全失；這是組學探勘最根本的挑戰，也是正則化、特徵選擇 與 多重比較校正 (BH-FDR) 成為標準配備的原因。

除單一 omics 之外，現代研究幾乎都需要 multi-omics integration：基因組 (WGS)、轉錄組 (bulk / single-cell RNA-seq)、表觀基因組 (ATAC / methylation)、蛋白組 (DIA-MS)、代謝組 (LC-MS) 各自有不同雜訊結構，必須先做批次校正 (batch correction) 才可能找到跨層次共變模式。

Typical transcriptomics: 20,000 genes × 50 samples, p/n = 400. OLS has no solution, Ridge over-fits, univariate t-tests largely die after multiple-testing correction. This is the founding challenge of omics mining and why regularization, feature selection, and multiple-testing control (BH-FDR) are mandatory.

Beyond a single omics, modern studies need multi-omics integration: genome (WGS), transcriptome (bulk / single-cell RNA-seq), epigenome (ATAC / methylation), proteome (DIA-MS), metabolome (LC-MS). Each has its own noise structure — batch correction is the first checkpoint before any cross-layer pattern can emerge.

⚠️
批次效應 ≈ 真實生物訊號：若樣本批次與表型混淆 (confounded)，再強的 ComBat / Harmony 都救不回。實驗設計時就要 randomize batch ↔ condition。 Batch effect ≈ real biology: if batch confounds phenotype, no ComBat / Harmony can rescue it. Randomize batch ↔ condition at experimental design.

LASSO 正則化路徑：λ 如何收縮係數

調整 λ (log scale)，觀察非零特徵數目下降、係數逐個被「壓到 0」。這就是 LASSO 同時做正則化與變項選擇的核心機制。

Adjust λ (log scale); watch the non-zero feature count drop as coefficients are squeezed to zero. This is how LASSO performs regularization and variable selection simultaneously.

多組學整合

二、多組學整合策略

➕

Early

把多 omics 直接 cbind；簡單但維度爆炸、忽略層次結構。

Concatenate features; simple but blows up dimension and ignores layer structure.

🔀

Intermediate

學共享潛在因子：MOFA+ / DIABLO / iCluster+ / scVAE。

Learn shared latent factors: MOFA+ / DIABLO / iCluster+ / scVAE.

🧩

Late

各 omics 各訓模型 → 在預測層融合（stacking / weighted vote）。

Train per-omics models, fuse at prediction (stacking / weighted vote).
