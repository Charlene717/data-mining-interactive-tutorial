CHAPTER 12 / 20 · MODULE C

序列模式探勘

關聯規則忽略「順序」——但臨床事件、購買行為、網頁瀏覽都是有時序的。本章介紹 GSP、SPADE、PrefixSpan、CloSpan，並帶入流程探勘 (process mining) 的橋樑。

Association rules ignore order, yet clinical events, purchases, and clickstreams are sequential. This chapter covers GSP, SPADE, PrefixSpan, CloSpan and bridges into process mining.

概覽

序列 vs 項集

關聯規則的輸入是 {A, B, C} 的集合，順序無關。序列 (sequence) 則是 ⟨(A)(B, C)(D)⟩ 這樣的有序事件集合——每個括號是同一時刻的 itemset，順序代表時間。一個序列「支援」另一序列若後者是子序列（順序保持但可跳項）。

生物醫學上：clinical pathway（住院→檢查→診斷→處方→出院）、用藥順序（先 metformin → 加 insulin）、單細胞發育軌跡、ICD 共現的時序版本，全都是序列資料。

Association rules treat the input as an unordered set {A, B, C}. A sequence is an ordered list of itemsets like ⟨(A)(B, C)(D)⟩ — each bracket is an itemset at one timestamp, the order encodes time. One sequence "supports" another if the latter is a subsequence (order preserved, gaps allowed).

Biomedical examples: clinical pathways (admit → labs → diagnosis → meds → discharge), medication sequences (metformin first, insulin added later), single-cell developmental trajectories, time-ordered ICD code co-occurrences.

演算法

一、四大序列演算法

演算法策略特色 / 缺點
GSP (1996)逐層搜尋 + 時序限制直觀；候選爆炸；可加 sliding window / min-gap / max-gapIntuitive; candidate explosion; supports sliding window / min-gap / max-gap
SPADE (2001)垂直 id-list透過 id-list 交集計算支援度，效率比 GSP 高Support via id-list intersection — faster than GSP
PrefixSpan (2001)pattern-growth + projection不產候選；長序列上效率最佳；最常用No candidate generation; best on long sequences; most popular
CloSpan (2003)只保留 closed 序列輸出大幅壓縮、無資訊損失Vastly fewer patterns, lossless
BIDE另一 closed 演算法記憶體更省Lower memory than CloSpan
💡
PrefixSpan 的核心：取一個常見前綴（如 ⟨A⟩），把所有序列「投影」成 A 之後的尾巴（投影資料庫），再對投影資料庫遞迴找頻繁前綴——所以叫 pattern-growth。不用枚舉候選，直接成長。

 PrefixSpan's core: take a frequent prefix (e.g., ⟨A⟩), project every sequence to whatever comes after A (the projected DB), then recursively mine frequent prefixes there — hence pattern-growth. No candidates: patterns are grown directly.

流程探勘

二、流程探勘：序列的圖形化升級

序列模式探勘輸出「常見子序列」清單，流程探勘進一步輸出可執行的工作流程圖 (Petri net / BPMN)。輸入也是 event log（每筆 = case_id, activity, timestamp），但輸出有並行、選擇、循環等結構，更接近真實流程。

三大發現演算法：

  • Alpha-algorithm：從 event log 直接推導 Petri net；對雜訊敏感。
  • Inductive Miner：以分治遞迴拆解 log，保證輸出合理 (sound) 的流程；對雜訊較穩健。
  • Heuristic Miner：以依賴度閾值剪枝，可處理含噪 log。

常用工具：Python 的 pm4py、ProM、Disco、Celonis。生醫應用：急診流程瓶頸分析、術後恢復路徑、與 clinical guideline 的順從度檢驗 (conformance checking)。

Sequential mining outputs a list of frequent subsequences. Process mining goes further: it outputs an executable workflow graph (Petri net / BPMN). Input is an event log (case_id, activity, timestamp), but output has parallelism, choice, and loops — much closer to real-world workflows.

Three discovery algorithms:

  • Alpha-algorithm: derives Petri nets directly from event log; noise-sensitive.
  • Inductive Miner: divide-and-conquer recursion, guarantees a sound model; noise-robust.
  • Heuristic Miner: prunes by dependency threshold, handles noisy logs.

Common tools: Python pm4py, ProM, Disco, Celonis. Biomedical uses: ED throughput bottlenecks, post-op recovery pathways, conformance checking against clinical guidelines.

決策樹

三、方法選擇決策樹

🌳 序列 / 流程探勘決策樹

Q1:
資料順序不重要、純共現？ → Ch 11 關聯規則 (Apriori)
Q2:
序列短、想看每一步候選？ → GSP（教學/小資料）
Q3:
序列長、長度差大、想直接 pattern-growth？ → PrefixSpan（現代標準）
Q4:
記憶體受限、稀疏序列？ → SPADE（垂直 id-list 交集）
Q5:
輸出爆炸需壓縮？ → CloSpan / BIDE（closed 序列）
Q6:
想要可視化的工作流程圖、檢查 conformance？ → pm4py / Inductive Miner
Q1:
Order doesn't matter, only co-occurrence? → Ch 11 Association Rules (Apriori)
Q2:
Short sequences, want to inspect candidate generation? → GSP (teaching / small data)
Q3:
Long / variable-length sequences, want direct pattern-growth? → PrefixSpan (modern default)
Q4:
Memory-tight, sparse sequences? → SPADE (vertical id-list)
Q5:
Output explosion — compress losslessly? → CloSpan / BIDE (closed sequences)
Q6:
Want a visual workflow + conformance checking? → pm4py / Inductive Miner
⚠️
序列探勘 vs 流程探勘：序列探勘列舉常見子序列，但不重構流程；流程探勘合成可執行流程圖（含並行、選擇），可做 conformance checking 與 bottleneck analysis。互補使用。 Sequence vs process mining: sequence mining enumerates frequent subsequences but doesn't reconstruct a workflow; process mining synthesizes an executable graph (with parallel splits and choices), enabling conformance and bottleneck analysis. Complementary tools.
