異常偵測 Anomaly Detection

「找出那些不該出現的點」——詐欺交易、敗血症惡化、儀器故障、罕病基因表現。

"Find the points that shouldn't be there" — fraud, sepsis deterioration, instrument fault, rare-disease expression.

概覽

什麼是異常？三種類型

異常 (anomaly / outlier) 是「顯著偏離整體分布」的觀測。Hawkins (1980) 的定義：「一個與其他觀測差異大到讓人懷疑它由不同機制產生的點」。在生物醫學上，這通常代表 儀器錯誤、罕見疾病、不良事件，或一個重要訊號的前兆（如 ICU 病人惡化）。

異常依結構分三類：(1) 點異常 (Point)——單筆觀測本身極端，如 HbA1c=20；(2) 情境異常 (Contextual)——只在特定情境下異常，如夏季氣溫 5°C；(3) 集體異常 (Collective)——個別正常但組合異常，如 ECG 突然規律的平坦段 (asystole)。

An anomaly / outlier is an observation that deviates markedly from the bulk. Hawkins (1980): "an observation that differs so much from others that it appears generated by a different mechanism." In biomedicine this often means instrument error, rare disease, adverse event, or an early warning of deterioration.

Three structural types: (1) Point — a single extreme record (HbA1c=20); (2) Contextual — anomalous only in context (5°C in summer); (3) Collective — individually normal but combined anomalous (a flat ECG segment indicating asystole).

演算法家族

一、五大偵測方法家族

📊

① 統計型 Statistical

Z-score、modified Z (MAD)、IQR (1.5×)、Grubbs、Mahalanobis（多變量+共變異）。簡單、可解釋，但假設分布。

Z-score, modified Z (MAD), IQR (1.5×), Grubbs, Mahalanobis (multivariate + covariance). Simple, interpretable, but assumes a distribution.

📏

② 距離型 Distance

KNN distance score：到第 k 個鄰居的距離越大越異常。對全域結構敏感、易受維度詛咒。

KNN distance score — the larger the distance to the k-th neighbor, the more anomalous. Captures global structure; suffers from curse of dimensionality.

🌌

③ 密度型 Density

LOF (Local Outlier Factor)：比較點的局部密度與鄰居的局部密度比值，>1 即異常。能處理變密度群集。

LOF (Local Outlier Factor) — ratio of a point's local density vs its neighbors'; >1 means anomalous. Handles varying-density clusters.

🌲

④ 隔離型 Isolation

Isolation Forest：隨機切割空間，異常點通常用更少切割就被隔離（路徑短）。O(n log n) 線性可擴展。

Isolation Forest — random partitioning; anomalies are isolated with fewer splits (shorter paths). O(n log n), highly scalable.

🧠

⑤ 學習型 Learning

One-Class SVM（決策邊界包絡）、Autoencoder（重建誤差 reconstruction error）、ECOD（empirical CDF, 無參、O(nd)）。

One-Class SVM (envelope boundary), Autoencoder (reconstruction error), ECOD (empirical CDF, parameter-free, O(nd)).

🔬

⑥ PyOD 2.0 / 3

PyOD 2.0 (2024) 加入 12 個深度模型與 LLM-based 自動選擇；PyOD 3 (2025) 擴至 60+ 偵測器，涵蓋 tabular/time/graph/text/image。

PyOD 2.0 (2024) added 12 deep models + LLM-based selection; PyOD 3 (2025) expands to 60+ detectors across tabular/time/graph/text/image.

決策樹

二、如何挑選偵測器？

🌳 異常偵測選擇樹

Q1:
單變量、大致 常態分布？ → Z-score / IQR / Grubbs
Q2:
多變量、有共變異？ → Mahalanobis distance / Elliptic Envelope
Q3:
群集 密度差異大（local pockets）？ → LOF
Q4:
大規模、高維、混合連續+類別？ → Isolation Forest / ECOD
Q5:
有 純正常樣本（半監督）？ → One-Class SVM / Autoencoder
Q6:
影像 / 訊號 / 文本，非線性結構？ → Deep Autoencoder / VAE / PyOD 2.0 deep models
Q7:
不確定怎麼選？ → PyOD MetaOD / LLM-based selection (PyOD 2.0)
Q1:
Univariate, approximately normal? → Z-score / IQR / Grubbs
Q2:
Multivariate with covariance? → Mahalanobis / Elliptic Envelope
Q3:
Clusters of varying density? → LOF
Q4:
Large-scale, high-D, mixed types? → Isolation Forest / ECOD
Q5:
Have only-normal training samples (semi-supervised)? → One-Class SVM / Autoencoder
Q6:
Image / signal / text with non-linear structure? → Deep Autoencoder / VAE / PyOD 2.0 deep models
Q7:
Not sure? → PyOD MetaOD / LLM-based selection (PyOD 2.0)
💡
無監督 vs 半監督：無監督 (Isolation Forest, LOF, ECOD) 不需 label，假設異常稀少；半監督 (One-Class SVM, Autoencoder) 在「純正常」資料上訓練，偵測新偏離。監督式則需要標記異常——但通常異常太少。 Unsupervised vs semi-supervised: Unsupervised (Isolation Forest, LOF, ECOD) needs no labels and assumes anomalies are rare. Semi-supervised (One-Class SVM, Autoencoder) trains on "normal-only" data and flags deviations. Supervised needs labeled anomalies — usually too few to be practical.

from sklearn.neighbors import LocalOutlierFactor

# 1) sklearn IsolationForest（contamination = 估計異常比例）

⚠️
contamination 設太高？contamination 不是「真實異常比例」而是「我願意誤殺多少」。在敗血症 0.5% 場景設 contamination=0.1 會偽陽性爆量。建議從 0.01–0.05 嘗試 + ROC/PR 評估。 contamination too high? It's not "true anomaly rate" but "how much I'm willing to misflag". In sepsis (0.5%), setting contamination=0.1 floods false positives. Start at 0.01–0.05 and validate with ROC/PR curves.
